# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `G1`。
- 阶段6已完成。

## 目标
生成最终交付物清单（`_交付清单.md`），列出所有需要交付的文件和目录，确保交付内容完整、无遗漏。

## Action
- 扫描项目根目录，识别以下应交付内容：
  - 源代码（`backend/`、`frontend/`、`modules/` 等）
  - 配置文件（`AGENTS.md`、`data-model.json`、`.env.example` 等）
  - 文档（`README.md`、`docs/` 等）
  - 部署脚本（`scripts/`、`docker-compose.yml` 等）
- 排除不应交付的目录（`.ai/`、`.trae/`、`node_modules/`、`__pycache__/` 等）。
- 生成清单文件 `_交付清单.md`，格式为列表，标注每个文件/目录的用途。
- 将清单存入项目根目录。

## Assert
- 清单包含所有应交付文件。
- 清单不包含临时或敏感文件。

## Artefact
- `_交付清单.md`

## Analysis
- 检查是否有遗漏的重要文件（如数据库迁移脚本）。
- 若发现敏感文件（如 `.env`），提示用户排除。

## Adjust
- 如果用户要求包含某些临时文件（如日志），可手动添加。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "G2",
    "steps_completed": ["...", "G1"]
  }
  ```
- 提示用户：下一步应执行 G2。

## 干预
无。
