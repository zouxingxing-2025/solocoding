# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B1`。
- `B0.2` 已完成。

## 目标
分析数据模型中每个字段的语义（如标识、名称、数量、金额、日期等），为后续功能模板匹配做准备。

## Action
- 读取 `data-model.json`，遍历所有实体和字段。
- 根据字段名称和类型，自动推断语义标签：
  - `id`, `*_id` → `identifier`
  - `name`, `title`, `label` → `name`
  - `count`, `amount`, `quantity`, `*_count` → `count`
  - `price`, `total`, `amount`（货币） → `amount`
  - `created_at`, `updated_at`, `date` → `date`
  - `description`, `content` → `text`
  - `status`, `state`, `type` → `enum`
- 将语义标签附加到 `data-model.json` 的字段定义中（新增 `semantic` 属性）。
- 保存更新后的 `data-model.json`。

## Assert
- 所有字段都有语义标签（无法推断的标记为 `unknown`）。

## Artefact
- 更新后的 `data-model.json`（含语义标签）。

## Analysis
- 检查是否有语义歧义（例如 `amount` 可能是金额也可能是数量），标记为 `ambiguous` 并提示用户确认。
- 若用户后续修正，记录为经验。

## Adjust
- 如果自动推断错误，允许用户手动修改（通过后续步骤或直接编辑文件）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B2",
    "steps_completed": ["...", "B1"]
  }
  ```
- 提示用户：下一步应执行 B2。

## 干预
无（自动推断，用户可后续修正）。
