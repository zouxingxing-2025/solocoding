allowed-tools: Bash, Write
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `G3`。
- `G2` 已完成。

## 目标
删除所有中间文件（如临时日志、缓存、未使用的demo），只保留最终交付物，确保项目目录整洁。

## Action
- 删除以下目录和文件：
  - `.ai/tmp/`（临时文件）
  - `.ai/cache/`（缓存）
  - `.ai/reports/`（审查报告，除非用户要求保留）
  - `.ai/demos/`（除最终demo外）
  - `.ai/logs/`（日志）
  - `.ai/backup/`（备份）
- 删除项目根目录下的临时文件（如 `*.tmp`、`*.bak`）。
- 如果 `.ai/` 目录为空，则删除该目录。
- 记录删除操作日志（`.ai/logs/cleanup.log`，但该日志本身也会被删除）。

## Assert
- 所有中间文件已删除。
- 项目目录只保留最终交付物。

## Artefact
- 清理日志（可选，删除前输出）。

## Analysis
- 检查是否有误删重要文件（如 `.env` 配置），若有则恢复。
- 若磁盘空间紧张，提示用户手动清理大型缓存。

## Adjust
- 如果用户希望保留某些中间文件（如审查报告），可设置 `keep_reports: true`。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "G4",
    "steps_completed": ["...", "G3"]
  }
  ```
- 提示用户：下一步应执行 G4。

## 干预
无。
