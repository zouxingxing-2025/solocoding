---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `C4`。
- `C3` 已完成。

## 目标
若沙箱测试失败，自动重试（最多3次），若重试后仍失败，则回退到模块生成步骤（B9）或标记为"需人工介入"。

## Action
- 读取 `C3` 生成的测试报告（`.ai/reports/test_report.html`）和失败日志。
- 分析失败原因：
  - 若为环境问题（依赖缺失、配置错误），尝试自动修复后重试。
  - 若为代码逻辑问题，回退到 `B9` 重新生成模块代码。
- 记录重试次数（最多3次）。
- 若3次重试后仍然失败，将模块状态改为"需人工介入"，并生成详细错误报告。

## Assert
- 重试次数不超过3次。
- 回退后状态正确更新。

## Artefact
- 更新后的进度状态。
- 错误报告（`.ai/reports/retry_failed.log`）。

## Analysis
- 分析失败模式，若为常见问题（如超时、网络），可自动增加等待时间。
- 若同一模块反复失败，建议用户检查需求或数据模型。

## Adjust
- 若用户选择跳过，可标记为"已知问题"并继续。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "C5",
    "steps_completed": ["...", "C4"]
  }
  ```
- 提示用户：下一步应执行 C5。

## 干预
若重试后仍失败，需用户确认是否继续。
