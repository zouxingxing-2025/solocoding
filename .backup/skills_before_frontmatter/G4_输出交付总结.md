# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `G4`。
- `G3` 已完成。

## 目标
输出交付总结报告（`_交付总结.md`），包含项目概述、功能列表、使用说明、已知问题等，供用户参考。

## Action
- 读取项目元数据（`AGENTS.md`、`data-model.json`、`.ai/进度状态.json`）。
- 生成交付总结，包含以下章节：
  - 项目概述：名称、版本、简介。
  - 功能模块清单：从 `AGENTS.md` 第6章提取。
  - 技术栈：从 `AGENTS.md` 第2章提取。
  - 部署说明：如何启动、配置、访问。
  - 已知问题：如有未解决的缺陷，列出。
  - 后续建议：优化方向、扩展建议。
- 将总结写入 `_交付总结.md`，存入项目根目录。

## Assert
- 总结包含所有必需章节。
- 信息准确，与项目状态一致。

## Artefact
- `_交付总结.md`

## Analysis
- 检查是否有未记录的已知问题（如测试覆盖率不足）。
- 若用户提供了额外反馈，补充到总结中。

## Adjust
- 如果用户不需要总结，可跳过此步骤。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "G5",
    "steps_completed": ["...", "G4"]
  }
  ```
- 提示用户：下一步应执行 G5（交付完成）。

## 干预
无。
