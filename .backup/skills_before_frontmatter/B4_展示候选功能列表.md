# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B4`。
- `B3` 已完成。

## 目标
将匹配到的候选功能以清晰列表形式展示给用户，让用户勾选需要实现的功能。

## Action
- 读取 `.ai/tmp/candidate_functions.json`，获取候选功能列表。
- 向用户展示候选功能（分组展示），并提问："请勾选您需要实现的功能（可多选）。您也可以跳过，稍后手动添加。"
- 等待用户回复，记录勾选的功能ID列表到 `.ai/进度状态.json` 的 `selected_functions`。
- **【新增】优先级排序**：
  - 向用户提问："请为以下勾选的功能设定优先级（高/中/低），这将影响开发顺序。您也可以接受默认优先级（按业务价值排序）。"
  - 展示勾选的功能列表，每个功能旁边让用户选择优先级（高/中/低）。
  - 记录优先级到 `.ai/进度状态.json` 的 `function_priorities` 字段（格式：`[{id: "mod_001", priority: "high"}, ...]`）。
- 如果用户未指定优先级，AI 根据以下规则自动排序：
  - 核心业务流程优先（如"岗位发布"优先于"报表导出"）
  - 依赖关系：被其他功能依赖的优先
  - 用户明确强调的优先
- 将优先级信息写入 `function_priorities`。

## Assert
- 用户至少勾选一个功能（或选择跳过）。

## Artefact
- 进度状态中的 `selected_functions` 列表。

## Analysis
- 如果用户未勾选任何功能，提示可能无法生成有意义的应用。

## Adjust
- 如果用户没有勾选任何功能，提示至少选择一个，或允许跳过（后续手动添加）。
- 如果用户对优先级排序有异议，可手动调整。

## Advance
- 更新进度状态后，进入 `B5`（收集自定义功能）。

## 干预
用户需勾选功能或确认跳过。
