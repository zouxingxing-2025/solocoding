# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A5`。
- `A4` 已完成。

## 目标
为每个实体提取字段及其类型。

## Action
- 遍历 `entities` 列表，从需求文档（`workflow_desc`、`input_pages`、`output_pages`）中提取相关字段。
- 推断字段类型（string, integer, float, date, boolean, reference）。
- 为每个实体生成字段列表，存入进度状态的 `fields` 字段（格式：`{ entity: [{ name, type, required }] }`）。

## Assert
- 每个实体至少有一个字段（通常包括 ID 或主键）。

## Artefact
- 进度状态中的 `fields` 结构。

## Analysis
- 检查是否有字段名重复或语义模糊。
- 识别可能的主键（如 `id`、`{entity}_id`）。

## Adjust
- 如果某实体字段为空，提示用户补充或从常见模板中自动添加（如 `id`, `created_at`）。
- 若字段类型无法确定，使用 `string` 作为默认。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A6",
    "steps_completed": ["...", "A5"]
  }
  ```
- 提示用户：下一步应执行 A6。

## 干预
无。
