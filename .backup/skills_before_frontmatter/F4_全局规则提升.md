---
name: 全局规则提升
description: 将验证次数超过阈值的经验提升为全局规则，应用到所有项目。
allowed-tools: Read, Write
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F4`。
- `F3` 已完成（经验库已去重）。

## 目标
将验证次数超过阈值（如3次）的经验提升为全局规则，写入全局经验库，并更新核心契约模板。

## Action
- 扫描 `.ai/经验库/索引.json`，筛选验证次数 >= 3 的条目。
- 对于每个候选条目：
  - 读取条目内容，提取可泛化的规则（如“所有Python项目必须使用类型提示”）。
  - 检查全局经验库（`~/.ai_experience/global-rules/`）是否已存在相同规则。
  - 若不存在，生成全局规则文件（YAML格式），包含规则ID、描述、适用技术栈、约束条件、来源项目列表。
  - 将规则写入 `~/.ai_experience/global-rules/<category>/<rule_id>.yaml`。
- 更新全局经验库索引（`~/.ai_experience/index.json`）。
- 更新核心契约模板（`AGENTS.md` 的第0章），添加全局规则引用说明。
- 输出规则提升报告（`.ai/reports/promotion_report.md`）。

## Assert
- 符合条件的经验已提升为全局规则。
- 全局规则文件格式正确，索引已更新。

## Artefact
- 新增的全局规则文件（`~/.ai_experience/global-rules/` 下）。
- 更新后的全局索引。
- 规则提升报告。

## Analysis
- 分析哪些类型的经验更容易被提升（如安全、性能），用于优化自动记录策略。
- 若规则提升后导致现有项目冲突，记录为潜在问题。

## Adjust
- 如果用户手动指定某条经验必须/不得提升为全局规则，遵循用户指令。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "F5",
    "steps_completed": ["...", "F4"]
  }
  ```
- 提示用户：下一步应执行 F5（跨项目引用）。

## 干预
用户可手动干预规则提升决策。
