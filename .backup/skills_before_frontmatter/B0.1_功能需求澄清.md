# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B0.1`。
- 阶段1已完成，用户已勾选候选功能列表。

## 目标
对每个勾选的功能进行详细的需求澄清，包括输入、输出、边界条件、验收标准等，输出功能需求文档到 `specs/features/功能名/需求.md`。

## Action
- 读取用户勾选的功能列表（从 `.ai/进度状态.json` 的 `selected_functions` 获取）。
- 对于每个功能：
  - 向用户提问："请描述功能【功能名】的具体需求：1）输入是什么？（数据来源、格式）2）输出是什么？（展示形式、数据字段）3）有哪些边界条件？（如空值、极值、权限）4）验收标准是什么？（如何判断功能完成）"
  - 将用户回答整理为结构化文档。
  - 写入 `specs/features/{功能名}/需求.md`。
- 如果用户描述不完整，使用引导式提问补充。

## Assert
- 每个功能都有对应的需求文档。
- 需求文档包含输入、输出、边界条件、验收标准四部分。

## Artefact
- `specs/features/{功能名}/需求.md` 文件。

## Analysis
- 检查需求是否与数据模型一致（字段存在性）。
- 若发现需求中引用了数据模型中不存在的字段，标记为"待补充"。

## Adjust
- 如果用户无法描述，提供同类功能模板供参考。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B0.2",
    "steps_completed": ["...", "B0.1"]
  }
  ```
- 提示用户：下一步应执行 B0.2。

## 干预
用户需回答每个功能的需求。
