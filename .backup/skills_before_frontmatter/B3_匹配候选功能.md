# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B3`。
- `B2` 已完成。

## 目标
基于字段语义和功能模板，匹配出当前数据模型可以实现的候选功能列表。

## Action
- 读取 `.ai/tmp/function-templates-filtered.yaml` 和 `data-model.json`（含语义标签）。
- 遍历每个功能模板，检查其 `required_fields` 是否在数据模型中有对应字段（按语义匹配，如 `count` 语义字段可匹配任意计数类字段）。
- 对于匹配成功的模板，生成候选功能条目，包含：
  - 模板 ID 和名称
  - 所需字段映射（模板字段 → 实际实体.字段）
  - 计算表达式（替换为实际字段名）
  - 推荐的前端展示形式（KPI卡片、表格、图表）
- 将候选功能列表存入 `.ai/tmp/candidate_functions.json`。

## Assert
- 至少有一个候选功能（否则提示用户需求可能过于简单或数据模型不完整）。

## Artefact
- `.ai/tmp/candidate_functions.json`

## Analysis
- 对于部分匹配的模板（缺少某个非关键字段），标记为"部分匹配"，仍作为候选但降低优先级。
- 记录常见匹配失败原因（如缺少语义标签），用于优化语义推断。

## Adjust
- 若无任何候选功能，提示用户可能需要添加更多字段或自定义功能。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B4",
    "steps_completed": ["...", "B3"]
  }
  ```
- 提示用户：下一步应执行 B4。

## 干预
无（自动匹配）。
