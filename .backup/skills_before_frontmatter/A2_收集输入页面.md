# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A2`。
- `A1` 已完成。

## 目标
明确用户需要哪些输入页面（表单），每个页面包含哪些字段、校验规则和提交动作。

## Action
- 向用户提问："请列出您需要的输入页面（例如：发布岗位表单、候选人录入）。对于每个页面，告诉我：1）需要填写哪些字段？2）有哪些校验规则（如必填、数字范围）？3）提交后触发什么动作（如保存数据、发送通知）？"
- 将用户回答结构化为 JSON 格式，存入 `.ai/进度状态.json` 的 `input_pages` 字段。
- 如果用户描述模糊，提供常见示例（如"岗位发布表单"的字段清单）。

## Assert
- 每个输入页面至少有名称和字段列表。

## Artefact
- 进度状态中的 `input_pages` 字段。

## Analysis
- 识别字段类型（文本、数字、日期等），为后续数据模型做准备。

## Adjust
- 如果用户提到"类似XX系统"，可自动参考常见模板。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A3",
    "steps_completed": ["...", "A2"]
  }
  ```
- 提示用户：下一步应执行 A3。

## 干预
用户需描述输入页面。
