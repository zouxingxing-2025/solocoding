---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F2`。
- `F1` 已完成（经验条目已生成）。

## 目标
将待整理的经验条目进行结构化存储，包括分类、打标签、更新索引，以便后续检索和提升。

## Action
- 扫描 `.ai/experience-pending/` 目录下的所有经验条目文件。
- 对每个条目：
  - 根据内容自动判断类型（成功经验/失败教训）。
  - 根据涉及的技术栈（前端、后端、数据库、部署等）和领域（字段语义、模块选择、插件配置、测试等）进行分类。
  - 生成标签（如 `python`, `sql-injection`, `ui-ux`）。
- 将分类后的经验条目移动到 `.ai/experience-classified/` 下的对应子目录（如 `backend/api-design/`）。
- 更新全局索引（`.ai/experience/索引.json`），记录每个条目的元数据（路径、类型、标签、时间戳、验证次数初始为1）。
- 如果条目已存在相似内容（通过标题或标签相似度判断），则合并或标记重复。

## Assert
- 所有待整理经验已分类。
- 索引文件已更新，无重复或遗漏。

## Artefact
- 移动后的经验条目文件。
- 更新后的索引文件。

## Analysis
- 对于无法自动分类的条目，标记为“待人工分类”，并提示用户。
- 记录分类过程中发现的常见模式（如大量后端安全经验），可用于后续优化。

## Adjust
- 如果用户设置了自定义分类规则，优先遵循。
- 若分类冲突，按最细粒度分类。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "F3",
    "steps_completed": ["...", "F2"]
  }
  ```
- 提示用户：下一步应执行 F3（去重合并）。

## 干预
如果存在无法自动分类的条目，需用户手动分类。
