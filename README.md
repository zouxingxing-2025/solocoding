# AI 编程组件包

> 一套面向小白的、基于反向收敛和对抗性审查的 AI 编程工具包。适用于 TRAE SOLO 模式。

## 简介

本组件包提供了一套完整的、可执行的 AI 编程方法论，将软件开发过程分解为 86 个原子步骤，每个步骤对应一个技能文件（位于 `.trae/skills/`）。用户只需按顺序执行技能，AI 会自动引导完成从需求分析到部署交付的全过程。

**核心理念**：数据驱动、确定性优先、半自动人机协同、模块化即插即用、渐进式披露、自动化经验沉淀。

## 目录结构
```
项目根目录/
├── .trae/
│   └── skills/          # 86个原子步骤技能文件
├── .ai/                 # AI 工作目录（动态生成，可忽略）
│   └── config/          # 配置文件（技术栈、页面需求等）
├── shared/              # 共享资源
│   └── schemas/         # JSON Schema
├── scripts/             # 清理脚本
├── AGENTS.md            # 核心契约（不可修改）
├── data-model.json      # 数据模型（动态生成）
└── README.md            # 本文件
```

## 快速开始

### 1. 环境要求
- TRAE IDE（已安装并登录）
- 网络连接（用于下载依赖和组件）

### 2. 初始化项目
将本组件包解压到空项目目录，然后用 TRAE 打开该目录。

### 3. 启动 SOLO 模式
在 TRAE 中切换到 SOLO 模式（通常在主界面左下角）。

### 4. 执行第一个技能
在 SOLO 模式中输入：
```
请执行 P1_解压组件包.md
```
AI 会按顺序引导您完成后续步骤。您只需回答问题、做出选择。

### 5. 人工干预点
整个流程中只有 4 个人工干预点：
- A8：确认数据模型
- B4：勾选功能列表
- C8：选择最终 demo
- E4：提交变更需求

其他步骤全自动。

## 核心契约

`AGENTS.md` 是项目的唯一行为准则，AI 必须严格遵守。它定义了：
- 核心理念与胶水编程规则
- 对抗性审查规则（包括双角色 KPI、审查标准）
- 模块生命周期管理
- 清理规则
- 全局经验库使用规范

**重要**：请勿手动修改 `AGENTS.md`（除非您非常清楚后果）。所有动态配置通过 `.ai/config/` 下的 YAML 文件管理。

## 常见问题

### Q: 我没有任何编程经验，能用吗？
A: 能。您只需要用自然语言回答问题（如"请描述您的业务流程"），AI 会引导您完成。

### Q: 技能执行顺序是怎样的？
A: AI 会自动按阶段顺序执行（P1 → P2 → … → G5）。您只需说"继续"或执行下一个技能文件名。

### Q: 出错了怎么办？
A: AI 会自动重试或回退。如果问题持续，请将错误信息提供给开发者。

### Q: 如何更新技能包？
A: 用新版本覆盖 `.trae/skills/` 目录即可。已生成的项目文件不受影响。
