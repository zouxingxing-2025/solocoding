# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `G5`。
- `G4` 已完成。

## 目标
提示用户项目已准备就绪，可以交付。清理最后的临时文件（如进度状态备份），并输出完成信息。

## Action
- 删除 `.ai/backup/` 目录（如果存在且为空）。
- 删除 `.ai/进度状态.json.bak` 等备份文件。
- 向用户输出成功消息：

🎉 项目交付完成！

最终代码位于项目根目录。

交付总结已生成：_交付总结.md

您可以将项目打包或直接部署。
感谢使用 AI 编程组件包！

- 可选：自动打开交付总结文件。

## Assert
- 所有临时文件已清理。
- 用户已收到完成通知。

## Artefact
- 无（仅通知）。

## Analysis
- 检查是否有遗留的敏感文件（如 `.env`），提示用户手动处理。
- 记录项目总耗时和生成代码量，用于后续优化。

## Adjust
- 如果用户需要保留某些临时文件（如日志），可跳过删除步骤。

## Advance
- 标记整个流程结束，进度状态中 `is_completed = true`。

## 干预
无。


## When to Use This Skill
Use this skill when G5_交付完成 is required, such as during phase G tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
