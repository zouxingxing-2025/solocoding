# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E4`。
- `E3` 已完成（监控已配置，或用户选择跳过）。

## 目标
用户提交对已上线功能的变更需求（新增、修改、删除）。AI 记录变更请求，并进入影响分析阶段。

## Action
- 向用户提问："请描述您希望进行的变更：1）新增什么功能？2）修改现有功能的哪些部分？3）删除哪些功能？请尽量具体。"
- 将用户回答结构化，存入 `.ai/变更请求/current.md`，包含：
  - 变更描述
  - 预期效果
  - 优先级（高/中/低）
- 如果用户描述不清晰，追问具体细节（如"修改哪个字段？新功能的数据来源是什么？"）。

## Assert
- 变更请求已记录，包含必要信息。

## Artefact
- 变更请求文件（`.ai/变更请求/current.md`）。

## Analysis
- 识别变更类型（功能新增、修改、删除）。
- 评估是否需要紧急处理（如安全漏洞修复）。

## Adjust
- 如果用户没有变更需求，可直接跳过（标记 `no_change_requested: true`）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E5",
    "steps_completed": ["...", "E4"]
  }
  ```
- 提示用户：下一步应执行 E5（影响范围分析）。

## 干预
用户需提供变更需求描述。


## When to Use This Skill
Use this skill when E4_用户提交变更需求 is required, such as during phase E tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
