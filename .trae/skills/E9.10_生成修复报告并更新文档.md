# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E9.10`。
- `E9.9` 已完成（用户已确认修复成功）。

## 目标
生成缺陷修复的完整报告，包括问题描述、根因、修复方案、测试结果、用户确认等，并更新相关项目文档（如需求文档、技术方案、变更日志）。

## Action
- 收集本次缺陷修复过程中的所有文档：
  - 复现步骤（`E9.1`）
  - 根因分析（`E9.3`）
  - 修复方案（`E9.4`）
  - 修复日志（`E9.6`）
  - 回归测试（`E9.7`）
  - 全量测试报告（`E9.8`）
  - 用户验证确认（`E9.9`）
- 生成最终修复报告（`.ai/缺陷报告/当前缺陷/修复报告_<timestamp>.md`），包含：
  - 缺陷ID（自动生成或用户提供）
  - 标题和描述
  - 严重等级
  - 影响版本
  - 修复版本（当前项目版本）
  - 根因总结
  - 修复内容摘要（修改的文件和逻辑）
  - 测试结果（全量测试通过率、回归测试结果）
  - 用户确认记录
- 更新项目文档：
  - 若修复涉及需求变更，更新 `specs/features/.../需求.md` 或 `docs/requirements/current.md`。
  - 若修复涉及技术方案变更，更新 `specs/features/.../技术方案.md`。
  - 更新 `CHANGELOG.md`，添加本次修复的条目。
- 将缺陷报告和修复经验存入全局经验库的 `defect-patterns/` 目录（如果该模式尚未记录）。
- 清理临时缺陷文件（保留报告，删除中间日志可选）。

## Assert
- 修复报告已生成。
- 项目文档已更新，与代码保持一致。

## Artefact
- 修复报告（`.ai/缺陷报告/当前缺陷/修复报告_*.md`）。
- 更新后的项目文档。
- 经验库条目（若新增）。

## Analysis
- 检查修复报告是否完整，可供日后追溯。
- 评估是否需将本次修复经验提升为全局规则（例如：若同一类型的缺陷出现超过3次）。

## Adjust
- 如果用户要求，可将修复报告发送到指定渠道（如邮件、钉钉）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "F1",
    "steps_completed": ["...", "E9.10"],
    "defect_fixed": true
  }
  ```
- 提示用户：缺陷修复流程结束。下一步进入经验沉淀阶段（F1）。

## 干预
无（自动完成，但用户可要求补充信息）。


## When to Use This Skill
Use this skill when E9.10_生成修复报告并更新文档 is required, such as during phase E tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
