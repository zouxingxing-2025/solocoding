# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E2`。
- `E1` 已完成。

## 目标
配置应用的日志收集系统（如 ELK、Loki 或简单文件轮转），确保日志可集中查看和检索。

## Action
- 读取 `AGENTS.md` 中的技术栈和部署配置。
- 根据项目规模选择日志方案：
  - 轻量级：配置日志文件轮转（`logrotate`），将日志输出到 `/var/log/myapp/`。
  - 企业级：生成 Filebeat 或 Promtail 配置，发送日志到 Elasticsearch 或 Loki。
- 生成配置文件（如 `logrotate.conf`、`filebeat.yml`）。
- 将配置文件上传到服务器相应目录。
- 重启日志服务（如 `systemctl restart filebeat`）。
- 记录配置步骤到 `.ai/logs/logging_setup.log`。

## Assert
- 日志文件正常生成或发送到中央日志系统。
- 日志轮转策略生效（如保留最近7天）。

## Artefact
- 日志配置文件。
- 设置日志。

## Analysis
- 检查日志是否包含关键信息（错误、请求、性能数据）。
- 若磁盘空间有限，设置日志最大保留量。

## Adjust
- 如果用户不需要集中日志，可跳过此步骤（标记 `logging_enabled: false`）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E3",
    "steps_completed": ["...", "E2"]
  }
  ```
- 提示用户：下一步应执行 E3。

## 干预
无。
