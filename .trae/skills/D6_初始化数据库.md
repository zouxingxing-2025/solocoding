# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D6`。
- `D5` 已完成（依赖已安装）。

## 目标
在目标服务器上初始化数据库：创建数据库、运行迁移脚本、导入种子数据（如果需要）。

## Action
- 通过 SSH 连接到服务器，切换到项目目录。
- 读取 `.ai/config/deployment.yaml` 中的数据库连接信息（主机、端口、用户名、数据库名）。
- 检查数据库是否已存在：
  - 若不存在，创建数据库（如 `CREATE DATABASE IF NOT EXISTS project_db;`）。
- 运行数据库迁移脚本：
  - 如果使用 ORM（如 Alembic、TypeORM）：执行迁移命令（`alembic upgrade head`、`typeorm migration:run`）。
  - 如果使用原始 SQL 迁移：按顺序执行 `database/migrations/` 下的 SQL 文件。
- 如果配置了种子数据（`database/seeds/`），执行种子脚本。
- 记录执行输出和错误日志。

## Assert
- 数据库创建成功。
- 所有迁移脚本执行无错误。
- 种子数据（若有）成功导入。

## Artefact
- 数据库初始化日志（`.ai/logs/db_init.log`）。

## Analysis
- 检查迁移脚本是否与当前数据模型（`data-model.json`）一致。
- 若迁移失败，分析错误原因（如字段冲突、外键约束）。

## Adjust
- 如果数据库已存在且非空，询问用户是否覆盖或执行增量迁移（默认增量）。
- 若迁移脚本有顺序依赖，自动排序后执行。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D7",
    "steps_completed": ["...", "D6"]
  }
  ```
- 提示用户：下一步应执行 D7。

## 干预
若需要覆盖已有数据库，需用户确认。

Use this skill when D6_初始化数据库 is required, such as during phase D tasks.
