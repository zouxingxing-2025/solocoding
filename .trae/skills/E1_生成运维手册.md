# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E1`。
- 阶段4已完成（应用已部署）。

## 目标
生成项目的运维手册（`OPS.md`），包含启动、停止、备份、恢复、日志查看、故障排查等操作指南。

## Action
- 读取 `AGENTS.md`、`.ai/config/deployment.yaml` 以及已生成的启动脚本。
- 生成运维手册，包含以下章节：
  - 环境要求（操作系统、依赖版本）
  - 启动服务：命令示例（`docker-compose up -d` 或 `systemctl start myapp`）
  - 停止服务：命令示例
  - 重启服务：命令示例
  - 查看日志：日志路径和查看命令（`tail -f /var/log/myapp.log`）
  - 备份数据库：备份命令（`pg_dump` 或 `mysqldump`）
  - 恢复数据库：恢复命令
  - 常见故障排查：端口冲突、依赖缺失、权限问题等
- 将手册写入 `OPS.md` 文件。

## Assert
- 手册包含所有必需章节。
- 命令示例准确可执行。

## Artefact
- `OPS.md`

## Analysis
- 检查是否与部署配置一致（如端口、路径）。
- 若手册中引用了未定义的环境变量，提示用户补充。

## Adjust
- 如果用户有特殊运维工具（如 Kubernetes），可调整手册内容。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E2",
    "steps_completed": ["...", "E1"]
  }
  ```
- 提示用户：下一步应执行 E2。

## 干预
无。

Use this skill when E1_生成运维手册 is required, such as during phase E tasks.

## Not For
