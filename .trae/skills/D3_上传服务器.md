---
name: 上传服务器
description: 将最终制品上传到目标服务器（生产或预发布环境）。
allowed-tools: Bash, Network, Write
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D3`。
- `D2` 已完成（制品已打包）。

## 目标
将最终制品上传到目标服务器（生产或预发布环境），准备部署。

## Action
- 读取 `.ai/config/deployment.yaml` 中的服务器配置（地址、用户名、认证方式）。
- 确认服务器连接方式（SSH Agent 转发优先，若失败则提示用户提供密码）。
- 使用 `scp` 或 `rsync` 将制品上传到服务器的指定目录（如 `/opt/app/`）。
- 记录上传进度和结果（成功/失败）。
- 若上传失败，重试最多3次，间隔5秒。

## Assert
- 制品已成功传输到服务器。
- 目标目录存在或已创建。

## Artefact
- 上传日志（`.ai/logs/upload.log`）。

## Analysis
- 检查服务器磁盘空间是否充足。
- 若上传速度过慢，提示用户检查网络。

## Adjust
- 如果 SSH Agent 未配置，提示用户手动输入密码或配置免密。
- 若目标目录不存在，自动创建（需有权限）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D4",
    "steps_completed": ["...", "D3"]
  }
  ```
- 提示用户：下一步应执行 D4。

## 干预
如果多次上传失败，需用户手动上传或检查服务器状态。
