# 生成模块代码

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 为 `D1`。
- 数据模型（`data-model.json`）、业务规则（`.ai/temp/business_rules.json`）、角色任务数据（`.ai/temp/roles_tasks_fields.json`）、界面原型（`.ai/previews/app_preview.html`）已确认。

## 目标
生成可部署的 uni-app 项目代码（包含前端APP和后台管理），集成权限控制、业务规则校验、数学美学设计令牌，并生成数据库迁移脚本和后端API。

## Action

### 1. 加载所有确认信息
- 读取 `data-model.json`、业务规则、角色任务数据、界面原型（作为视觉参考）。

### 2. 生成数据库迁移脚本
- 根据数据模型，生成 SQL 建表语句（`database/migrations/001_init.sql`）。
- 包含字段类型、约束、外键、索引。
- 根据业务规则，添加数据库级约束（如 `CHECK (gap_count >= 0)`）。

### 3. 生成后端 API 代码（Node.js + Express 示例，可适配其他框架）
- 为每个实体生成标准 CRUD 接口（列表、详情、创建、更新、删除）。
- 集成权限中间件：根据权限矩阵，限制不同角色对 API 的访问。
- 集成业务规则校验：在创建/更新时校验规则（如手机号格式、唯一性）。
- 生成 API 文档（OpenAPI 格式）。

### 4. 生成前端代码（uni-app）
- **用户端（APP）**：
  - 岗位浏览页面（列表、筛选、详情）
  - 投递表单（集成规则校验）
  - 我的投递页面
- **管理端（Web）**：
  - 岗位管理页面（列表、创建、编辑、删除）
  - 候选人管理页面（列表、详情、状态变更）
  - 审批页面（待审批列表、审批表单）
- 所有页面引用 `docs/design-tokens.css`，使用 CSS 变量。
- 高频操作按钮置于右下角（悬浮按钮），高频字段置于左上角。
- 表单集成前端规则校验（实时提示）。

### 5. 生成权限控制代码
- 前端菜单根据角色动态显示（HR 看到管理菜单，候选人看到投递页面）。
- 后端 API 使用装饰器或中间件检查权限。

### 6. 生成单元测试和集成测试
- 为关键业务规则编写测试用例（如"岗位缺口数不能为负数"）。
- 为 API 接口编写集成测试。

### 7. 输出项目结构
- 按照 uni-app 标准目录组织代码：
```
project/
├── backend/ # Node.js 后端代码
├── frontend/ # uni-app 前端代码
│ ├── pages/ # 页面组件
│ ├── components/ # 公共组件
│ ├── utils/ # 工具函数（规则校验、权限）
│ └── App.vue
├── database/ # 迁移脚本
├── docs/ # 设计令牌、API文档
└── .ai/ # 工具包配置
```

### 8. 代码输出与验证
- 将生成的代码写入项目目录。
- 运行 `npm install` 安装依赖。
- 启动开发服务器，验证页面是否正常显示。

## Assert
- 所有实体都有对应的数据库表和 API。
- 前端页面引用设计令牌，无硬编码样式。
- 权限控制生效（不同角色登录后看到不同菜单）。
- 业务规则校验生效（输入非法数据时提示错误）。

## Artefact
- 数据库迁移脚本
- 后端 API 代码
- 前端 uni-app 代码
- 单元测试和集成测试
- API 文档
