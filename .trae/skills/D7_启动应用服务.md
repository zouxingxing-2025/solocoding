---
name: 启动应用服务
description: 在目标服务器上启动应用服务（如 Docker 容器、systemd 服务）。
allowed-tools: Bash, Network
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D7`。
- `D6` 已完成（数据库已初始化）。

## 目标
在目标服务器上启动应用服务（如 Docker 容器、systemd 服务），确保服务正常运行。

## Action
- 通过 SSH 连接到服务器，切换到项目目录。
- 根据部署方式启动服务：
  - Docker Compose: `docker-compose up -d`
  - systemd: `systemctl start myapp`
  - 直接运行: `nohup python app.py &`
- 等待服务启动（最多60秒）。
- 检查进程是否存在（`ps aux | grep app`）。
- 记录启动日志（`.ai/logs/service_start.log`）。

## Assert
- 服务进程存在。
- 服务监听在预期端口（如 `netstat -tuln | grep 8000`）。

## Artefact
- 启动日志。

## Analysis
- 如果启动失败，分析错误原因（端口占用、配置文件错误、依赖缺失）。
- 若使用 Docker，检查容器状态（`docker ps -a`）。

## Adjust
- 若启动失败，尝试自动修复（如释放端口、修正配置）后重试，最多2次。
- 若仍失败，输出错误报告并提示用户手动介入。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D8",
    "steps_completed": ["...", "D7"]
  }
  ```
- 提示用户：下一步应执行 D8。

## 干预
如果服务反复启动失败，需用户手动检查环境。
