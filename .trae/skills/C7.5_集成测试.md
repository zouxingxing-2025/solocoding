# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `C7.5`。
- `C7` 已完成（预览环境已部署）。

## 目标
将所有已通过对抗审查的模块部署到统一的预览环境，运行跨模块的端到端集成测试，验证模块间接口、数据流、依赖关系是否正确。

## Action
- 读取 `.ai/config/modules.yaml`，获取所有已生成且状态为"已验收"或"待验收"的模块。
- 将各模块的代码合并到一个统一的项目壳中（若尚未合并，调用 D1 生成集成壳）。
- 启动集成环境（使用 `docker-compose` 或本地服务）。
- 运行预先定义的集成测试用例（基于模块间契约和用户流程）：
  - 测试模块A的API是否被模块B正确调用。
  - 测试数据在模块间的传递是否一致（如用户认证信息、订单数据）。
  - 测试前端页面之间的跳转和状态同步。
- 收集测试结果，生成集成测试报告（`.ai/reports/integration_test.md`）。

## Assert
- 集成测试报告包含每个测试用例的结果（通过/失败）。
- 若全部通过，标记集成测试状态为 `passed`。

## Artefact
- 集成测试报告（`.ai/reports/integration_test.md`）。

## Analysis
- 对于失败的测试，定位到具体模块和原因（接口不匹配、数据格式错误、依赖缺失）。
- 若失败原因可自动修复（如缺失环境变量），尝试修复并重新运行。

## Adjust
- 若发现模块间问题，将相关模块状态回退至"生成中"，返回阶段2修复。
- 若集成环境启动失败，提示用户检查资源或手动干预。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "C8",
    "steps_completed": ["...", "C7.5"]
  }
  ```
- 提示用户：下一步应执行 C8。

## 干预
如果集成测试失败且无法自动修复，需用户介入决策。


## When to Use This Skill
Use this skill when C7.5_集成测试 is required, such as during phase C tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
