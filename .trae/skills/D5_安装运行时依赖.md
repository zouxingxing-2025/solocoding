# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D5`。
- `D4` 已完成（制品已解压）。

## 目标
在目标服务器上安装项目所需的运行时依赖（如 Python 包、Node.js 模块、系统库）。

## Action
- 通过 SSH 连接到服务器，切换到项目目录。
- 根据技术栈执行依赖安装：
  - Python: 使用 `pip install -r requirements.txt`（建议使用虚拟环境）
  - Node.js: 使用 `npm install` 或 `yarn install`
  - Java: 使用 `mvn install` 或 `gradle build`
- 若需要系统级依赖（如 `libpq-dev`），使用包管理器安装（`apt-get install -y` 或 `yum install`）。
- 记录安装输出和错误日志。
- 若安装失败，重试最多2次，每次更换镜像源（如使用清华源、阿里云）。

## Assert
- 所有依赖安装成功，无报错。
- 关键命令（如 `python --version`、`node --version`）可执行。

## Artefact
- 依赖安装日志（`.ai/logs/deps_install.log`）。

## Analysis
- 检查是否有版本冲突，若存在则尝试自动降级或升级。
- 记录安装耗时，用于优化后续部署。

## Adjust
- 如果虚拟环境未创建，自动创建并激活。
- 若缺少系统包管理器权限，提示用户手动安装或使用容器。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D6",
    "steps_completed": ["...", "D5"]
  }
  ```
- 提示用户：下一步应执行 D6。

## 干预
如果依赖安装反复失败，需用户手动介入（如提供正确镜像源或安装包）。
