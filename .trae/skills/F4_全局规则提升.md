# F4_全局规则提升.md

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F4`。
- `F3` 已完成（经验去重合并已完成）。

## 目标
将经过验证的、具有普遍适用性的经验条目提升为全局规则，写入 `~/.ai_experience/global-rules/common-constraints.yaml`，并在后续项目的 `AGENTS.md` 模板中自动引用。

## Action
- 扫描 `.ai/经验库/已分类/` 下所有经验条目，筛选满足以下条件的：
  - 验证次数 ≥ 3（在不同项目中均有效）。
  - 无矛盾经验存在（即没有相反的建议）。
  - 属于可形式化的约束（如"禁止使用 eval()"、"所有 API 必须限流"）。
- 对每个候选规则，生成结构化规则条目：
  ```yaml
  - rule_id: "GEN_XXX"
    description: "规则描述"
    severity: "Critical | Major | Minor"
    applies_to: ["frontend", "backend", "database", "deployment"]
    enforcement: "必须 | 建议"
    examples:
      bad: "错误示例"
      good: "正确示例"
  ```
- 将新规则合并到 `~/.ai_experience/global-rules/common-constraints.yaml` 中（避免重复）。
- 更新全局经验库索引，标记该经验已被提升为全局规则。
- 生成提升报告（`.ai/reports/global_rules_promotion.md`），列出新增的规则。

## Assert
- 新规则已写入全局规则文件。
- 经验库中对应条目已添加 `promoted: true` 标记。

## Artefact
- 更新后的 `common-constraints.yaml`。
- 提升报告。

## Analysis
- 检查新规则是否与已有规则冲突，若冲突则提示用户解决。
- 评估规则的影响范围，避免过度约束。

## Adjust
- 如果用户不同意自动提升，可改为建议模式，由用户确认后写入。
- 若全局规则文件不存在，自动创建。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "F5",
    "steps_completed": ["...", "F4"]
  }
  ```
- 提示用户：下一步应执行 F5（跨项目引用）。

## 干预
若存在冲突或用户需审核，提供规则列表供用户确认。
