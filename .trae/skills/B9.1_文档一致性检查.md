# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B9.1`。
- `B9` 已完成。

## 目标
检查生成的代码是否与当前需求文档、数据模型、API设计保持一致。若不一致，自动修正代码或提示用户更新文档。

## Action
- 读取以下文件：
  - 当前模块的 `specs/features/*/需求.md` 和 `技术方案.md`
  - `data-model.json`
  - 已生成的代码文件（后端控制器、前端组件等）
- 逐项对比：
  - 需求文档中定义的输入字段、校验规则、提交动作是否在代码中实现。
  - 需求文档中定义的展示字段、聚合逻辑、筛选条件是否在代码中实现。
  - 数据模型中的字段是否与代码中的实体类/数据库迁移一致。
  - API 接口签名是否与技术方案一致。
- 生成差异报告（`.ai/reports/consistency_check.md`）。

## Assert
- 代码与文档之间无 Critical 级别的不一致（如缺失必填字段、错误的数据类型）。

## Artefact
- 差异报告（`.ai/reports/consistency_check.md`）。

## Analysis
- 对于发现的不一致，判断严重等级：
  - Critical：功能缺失、字段类型错误、API 接口缺失。
  - Major：命名不一致、非关键字段缺失。
  - Minor：注释、顺序差异。
- 若存在 Critical 问题，回退到 B9 重新生成对应模块。

## Adjust
- 自动修复 Minor 问题（如添加缺失的注释、调整代码顺序）。
- 对于 Major 问题，尝试自动修复；若无法修复，标记并提示用户。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B10",
    "steps_completed": ["...", "B9.1"]
  }
  ```
- 提示用户：下一步应执行 B10。

## 干预
如果存在无法自动修复的 Major 问题，需用户确认或手动调整。


## When to Use This Skill
Use this skill when B9.1_文档一致性检查 is required, such as during phase B tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
