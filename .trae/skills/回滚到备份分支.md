# 回滚到备份分支

## 前置条件
- 当前目录是一个 Git 仓库。
- 存在至少一个备份分支（命名以 `backup/` 开头）。

## 目标
列出所有备份分支（本地+远程），用户选择后，将当前分支完全重置到该备份分支的状态，并丢弃当前分支之后的所有提交。

## Action
1. 更新远程分支信息：`git fetch origin`
2. 获取所有本地和远程备份分支：
   - 本地：`git branch --list "backup/*"`
   - 远程：`git branch -r --list "origin/backup/*" | sed 's/origin\///'`
3. 合并去重，按时间戳倒序排列，展示给用户（最多10个）。
4. 用户选择分支（输入序号或完整分支名）。
5. 确认回滚操作（警告会丢失当前分支后续提交）。
6. 执行回滚：
   - 方法：`git reset --hard <备份分支名>`
   - 强制推送到远程：`git push --force-with-lease`
7. 输出回滚成功信息。

## Assert
- 回滚后，工作目录与备份分支完全一致。
- 远程仓库也被强制更新。

## Artefact
- 回滚后的代码状态。
- 回滚日志（`.ai/rollback_log.txt`）。

## Analysis
- 如果用户选择的分支不是当前分支的祖先，回滚会丢失一些提交，需再次确认。
- 如果用户选择的分支是当前分支的祖先，可以使用 `--ff-only` 更安全，但强制重置也可。

## Adjust
- 若用户未选择，默认回滚到最新的备份分支。

## Advance
- 回滚完成后，提示用户测试功能。

## When to Use This Skill
- 用户发现修改出错，需要恢复到之前状态时。
- AI 检测到关键步骤失败，建议用户回滚。

## Not For
- 仅查看备份列表而不实际回滚。
- 回滚到非备份分支。
