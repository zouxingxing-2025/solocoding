# F3_去重合并.md

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F3`。
- `F2` 已完成（经验已分类存储）。

## 目标
定期（如每项目结束或每月）对经验库中的条目进行去重和合并，删除过时经验，提升经验库质量。

## Action
- 扫描 `.ai/经验库/已分类/` 下所有经验条目。
- 计算条目之间的相似度（基于标题、标签、解决方案文本的语义相似度）。
- 若相似度 > 0.8，视为重复，执行合并：
  - 保留最早或最完整的条目。
  - 将其他条目的验证次数、出现项目等信息合并到保留条目中。
  - 删除重复条目，并更新索引。
- 对于超过2年未被引用的经验条目，标记为"过时"，并提示用户确认是否删除（默认移动到归档目录）。
- 输出去重合并报告（`.ai/reports/dedupe_report.md`）。

## Assert
- 重复条目已合并，索引已更新。
- 过时条目已处理。

## Artefact
- 更新后的经验库文件。
- 去重合并报告。

## Analysis
- 分析重复条目产生的常见原因（如同一问题在不同项目中以不同措辞记录），优化自动记录模板。
- 若合并过程中出现冲突（如矛盾的建议），保留更可靠的版本（验证次数多或来源项目更成熟）。

## Adjust
- 如果用户要求保留某些重复条目（如故意记录的不同视角），可设置白名单。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "F4",
    "steps_completed": ["...", "F3"]
  }
  ```
- 提示用户：下一步应执行 F4（全局规则提升）。

## 干预
若合并涉及重要经验，需用户确认。
