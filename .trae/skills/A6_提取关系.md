name: 提取关系
description: 提取实体间的关系（一对一、一对多、多对多）。
allowed-tools: Read, Write
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A6`。
- `A5` 已完成。

## 目标
提取实体间的关系（一对一、一对多、多对多）。

## Action
- 读取进度状态中的 `entities` 和 `fields`。
- 根据字段命名和业务逻辑推断关系：
  - 若实体A有字段 `entityB_id`，则 A 属于 B（多对一）。
  - 若存在中间表（如 `order_products`），则为多对多。
- 将关系存入进度状态的 `relations` 字段，格式：`{ from: "A", to: "B", type: "one-to-many", via: "fieldName" }`。

## Assert
- 关系定义清晰，无循环依赖。

## Artefact
- 进度状态中的 `relations` 列表。

## Analysis
- 检查外键引用是否指向存在的实体。
- 若发现未定义的外键字段，提示用户确认。

## Adjust
- 如果关系无法自动推断，向用户提问："实体A和实体B是什么关系？一对多？多对多？"

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A7",
    "steps_completed": ["...", "A6"]
  }
  ```
- 提示用户：下一步应执行 A7。

## 干预
无。
