# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B4`。
- `B3` 已完成。

## 目标
将匹配到的候选功能以清晰列表形式展示给用户，让用户勾选需要实现的功能。

## Action
- 读取 `.ai/tmp/candidate_functions.json`。
- 按业务主题分组（如"销售分析"、"用户管理"）展示。
- 每个功能显示：名称、描述、所需字段、计算表达式、推荐展示形式。
- 向用户提问："请勾选您需要实现的功能（可多选）。您也可以稍后自定义功能。"
- 等待用户勾选，将结果存入 `.ai/进度状态.json` 的 `selected_functions`。

## Assert
- 用户至少勾选一个功能（或选择跳过）。

## Artefact
- 进度状态中的 `selected_functions` 列表。

## Analysis
- 如果用户未勾选任何功能，提示可能无法生成有意义的应用。

## Adjust
- 若用户要求更多细节，可展开每个功能的详细说明。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B5",
    "steps_completed": ["...", "B4"]
  }
  ```
- 提示用户：下一步应执行 B5。

## 干预
用户需勾选功能或确认跳过。
