# E9.6_执行修复.md

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E9.6`。
- `E9.5` 已完成（用户已选择修复方案）。

## 目标
按照用户选择的修复方案，修改相关代码、配置文件或数据结构，并确保修复不引入新问题。

## Action
- 读取用户选择的修复方案（`.ai/缺陷报告/当前缺陷/修复方案.md` 中指定的方案）。
- 根据方案中的修改描述，逐一修改文件：
  - 代码文件：应用 diff 修改（如修正条件、添加检查、优化查询）。
  - 配置文件：修改环境变量、依赖版本、配置参数。
  - 数据结构：若需修改 `data-model.json`，同步更新并生成迁移脚本。
- 修改后，执行受影响的单元测试和集成测试（根据影响分析确定）。
- 若测试失败，尝试自动修正（如调整代码）或回滚修改。
- 记录修复日志（`.ai/缺陷报告/当前缺陷/修复日志.log`）。

## Assert
- 所有变更已成功应用。
- 相关测试通过（无新增失败）。

## Artefact
- 修改后的代码文件、配置文件。
- 修复日志。

## Analysis
- 检查修复是否引入新的代码风格或命名问题。
- 验证修复后的功能是否满足验收标准（可参考原需求文档）。

## Adjust
- 如果测试失败且无法自动修复，回滚所有变更，并返回 E9.4 重新生成方案。
- 若修复涉及数据结构变更，确保迁移脚本已生成并执行。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E9.7",
    "steps_completed": ["...", "E9.6"]
  }
  ```
- 提示用户：下一步应执行 E9.7（编写回归测试）。

## 干预
若修复失败且自动回滚后仍无法解决，需用户手动输入。
