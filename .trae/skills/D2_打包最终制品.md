# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D2`。
- `D1` 已完成（集成壳已生成）。

## 目标
将集成壳及其所有依赖打包成可部署的制品（如 Docker 镜像、压缩包、安装程序），便于上传到目标服务器。

## Action
- 读取集成壳代码目录，确定打包方式（根据 `AGENTS.md` 第2章技术栈和用户偏好）：
  - 若使用 Docker：生成 `Dockerfile`（如尚未生成），并构建 Docker 镜像（`docker build -t project-name:latest .`）。
  - 若使用传统部署：生成压缩包（`.tar.gz` 或 `.zip`），包含所有代码、依赖清单、启动脚本。
- 将制品存放到 `.ai/artifacts/` 目录，命名格式：`project-name-{timestamp}.{ext}`。
- 记录制品路径、大小、构建日志到 `.ai/进度状态.json`。

## Assert
- 制品生成成功，无错误。
- 制品大小合理（不应包含开发依赖或临时文件）。

## Artefact
- 制品文件（`.ai/artifacts/project-name-*.{ext}`）
- 构建日志（`.ai/logs/build.log`）

## Analysis
- 检查制品是否包含敏感信息（如 `.env` 文件、密钥），若有则排除或提示用户。
- 若 Docker 构建失败，回退到压缩包方式。

## Adjust
- 如果打包失败（如磁盘空间不足），提示用户清理或选择其他方式。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D3",
    "steps_completed": ["...", "D2"]
  }
  ```
- 提示用户：下一步应执行 D3。

## 干预
无。


## When to Use This Skill
Use this skill when D2_打包最终制品 is required, such as during phase D tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
