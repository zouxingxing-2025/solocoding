# 远程部署

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F4`。
- `F3` 已完成（部署包已上传）。

## 目标
远程部署应用到服务器，配置环境变量、启动服务，确保应用正常运行。

## Action
- 连接远程服务器（使用 SSH）。
- 解压部署包到指定目录。
- 配置环境变量（如数据库连接、端口等）。
- 启动服务：
  - 后端：`npm start` 或 `systemctl start app`
  - 前端：配置 Nginx 或其他 web 服务器
- 验证服务是否正常启动。

## Assert
- 服务启动成功，可通过 URL 访问。
- 数据库连接正常。

## Artefact
- 部署日志（`deploy/deploy.log`）。

## Analysis
- 检查服务启动时间，若过长，优化启动流程。

## Adjust
- 若部署失败，检查网络、权限、配置等问题后重试。

## Advance
- 更新 .ai/进度状态.json：
  ```json
  {
    "current_step": "F5",
    "steps_completed": ["...", "F4"]
  }
  ```
- 提示用户：下一步应执行 F5（健康检查）。

## 干预
- 无。
