# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D1`。
- `C9` 已完成（最终demo已锁定）。

## 目标
生成一个统一的应用集成壳（Shell），将所有已选中的功能模块组装成一个完整的、可运行的应用程序。集成壳负责统一路由、导航、认证、主题样式等全局功能。

## Action
- 读取 `.ai/config/modules.yaml` 获取所有最终模块列表。
- 读取最终demo代码（位于 `.ai/demos/final/` 或项目根目录）。
- 生成集成壳项目结构：
  - 后端：创建主入口文件（如 `main.py`、`app.js`），聚合各模块的路由和中间件。
  - 前端：创建根组件（如 `App.vue`、`App.jsx`），包含导航菜单、路由配置、全局状态管理。
  - 统一配置文件（如 `config.js`、`settings.py`），集中管理环境变量、API前缀等。
- 将各模块的代码复制或链接到集成壳的对应目录（如 `backend/modules/`、`frontend/views/`）。
- 生成启动脚本（`start.sh`、`start.ps1`）和 `docker-compose.yml`（如需要）。
- 保存集成壳代码到项目根目录（或指定输出目录）。

## Assert
- 集成壳包含所有功能模块的引用。
- 启动后能正常访问各模块页面和API。

## Artefact
- 集成壳代码（项目根目录下）。
- 启动脚本和配置文件。

## Analysis
- 检查模块间是否有命名冲突（如路由前缀、组件名）。
- 验证统一认证中间件是否已正确集成。

## Adjust
- 若发现冲突，自动重命名或提示用户手动解决。
- 若缺少公共依赖，自动在根目录的 `package.json` 或 `requirements.txt` 中添加。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D2",
    "steps_completed": ["...", "D1"]
  }
  ```
- 提示用户：下一步应执行 D2。

## 干预
无。
