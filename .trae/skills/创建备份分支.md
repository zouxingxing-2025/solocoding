# 创建备份分支

## 前置条件
- 当前目录是一个 Git 仓库（已有 GitHub 远程）。
- 当前分支干净（或用户确认可以暂存未提交变更）。

## 目标
在执行可能破坏项目的修改之前，创建一个基于当前分支的备份分支，命名规则为 `backup/YYYYMMDD-HHMMSS`，并推送到远程仓库。

## Action
1. 获取当前分支名：`git branch --show-current`
2. 生成时间戳（格式：`YYYYMMDD-HHMMSS`）：
   - Windows PowerShell: `Get-Date -Format "yyyyMMdd-HHmmss"`
   - Linux/macOS: `date +%Y%m%d-%H%M%S`
3. 创建备份分支：`git checkout -b backup/<时间戳>`
4. 将当前所有变更（包括未提交的）提交到备份分支：
   - `git add .`
   - `git commit -m "Backup before operation: <操作描述>"`
5. 推送到远程：`git push origin backup/<时间戳>`
6. 切回原分支：`git checkout <原分支名>`
7. 如果原分支有未提交的变更，它们会被带到备份分支，原分支应保持干净（可提示用户重新应用）。

## Assert
- 备份分支已成功创建并推送。
- 原分支状态不变（未提交的变更已移到备份分支）。

## Artefact
- 本地和远程备份分支。
- 记录最后创建的备份分支名到 `.ai/last_backup_branch.txt`。

## Analysis
- 如果推送失败（无权限、网络问题），提示用户手动推送或跳过远程备份。

## Adjust
- 若用户未提供操作描述，使用 "auto-backup"。

## Advance
- 备份完成后，继续执行原定的修改操作。

## When to Use This Skill
- 在执行任何可能破坏项目或需要可回滚的修改之前（如 `A9_应用修正.md`、`B9_生成模块代码.md`、`E6_重新生成受影响模块.md`）。
- 用户手动执行 `@创建备份分支` 以主动创建快照。

## Not For
- 临时性、不影响项目稳定性的小修改。
- 用户明确不需要备份的场景。
