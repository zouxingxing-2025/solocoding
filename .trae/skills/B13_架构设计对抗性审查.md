# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B13`。
- `B12` 已完成。

## 目标
由审查者Agent对架构设计进行对抗性审查，检查模块划分、接口定义、技术栈一致性、可扩展性、依赖合理性等。

## Action
- 审查者Agent（独立上下文）加载：
  - `AGENTS.md` 第1章（审查标准）
  - `.ai/config/modules.yaml`
  - `.ai/config/plugins.yaml`
  - 已生成的代码结构（目录树）
- 按以下维度检查：
  - 模块划分：是否高内聚低耦合？是否有循环依赖？
  - 接口定义：API 是否清晰、版本化？前后端契约是否一致？
  - 技术栈一致性：是否遵循 `AGENTS.md` 定义的技术栈？是否有不必要的混合技术？
  - 可扩展性：是否便于新增功能？是否预留扩展点？
  - 依赖合理性：模块间依赖是否最小化？是否有过度的全局状态？
- 生成问题列表（含严重等级、位置、建议修复方案）。
- 输出审查报告到 `.ai/reports/architecture_review.md`。

## Assert
- 审查报告已生成，包含所有检查维度。
- 若没有发现问题，报告仅包含"通过"。

## Artefact
- 审查报告（`.ai/reports/architecture_review.md`）。

## Analysis
- 匹配缺陷模式库中的架构反模式（如"循环依赖"、"上帝对象"）。
- 记录常见问题，用于优化架构模板。

## Adjust
- 若存在 Critical 问题，将架构设计状态回退至 B6，要求执行者修复。
- 若只有 Major/Minor，生成修复建议，由执行者决定是否修复（可通过用户终裁）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "C1",
    "steps_completed": ["...", "B13"]
  }
  ```
- 提示用户：阶段2完成，下一步进入阶段3（测试与验证），应执行 C1。

## 干预
如果审查者与执行者对问题严重等级有争议，提交用户终裁。


## When to Use This Skill
Use this skill when B13_架构设计对抗性审查 is required, such as during phase B tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
