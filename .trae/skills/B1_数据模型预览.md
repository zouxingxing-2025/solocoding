# 数据模型预览

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 为 `B1`。
- 数据模型草稿已生成：`.ai/tmp/data-model-draft.json`。

## 目标
将技术数据模型转换为业务语言描述和示例数据卡片，让小白客户直观地确认"需要记录哪些信息"。用户无需理解实体、字段、外键等技术概念。

## Action

### 1. 读取数据模型草稿
- 读取 `.ai/tmp/data-model-draft.json`（内部格式，包含实体、字段、类型、约束等）。

### 2. 转换为业务语言
- 遍历每个实体，根据字段名和语义，生成业务描述。
- 为每个实体生成一个示例数据卡片（使用模拟值）。

**示例输出**（展示给用户）：

```markdown
根据您的描述，我理解您的系统需要管理以下信息。请确认是否完整、正确：

**岗位信息**
- 岗位名称（例如：Java工程师）
- 工作城市（例如：上海）
- 需要招聘的人数（例如：3人）
- 状态（例如：招聘中、已关闭）

**候选人信息**
- 姓名
- 手机号（11位数字）
- 应聘岗位（从岗位列表中选择）

**投递记录**
- 哪个岗位（从岗位列表中选择）
- 哪个候选人（从候选人列表中选择）
- 投递时间（系统自动记录）

如果有遗漏或错误，请直接描述（例如："还需要记录岗位的薪资范围"）。
```

### 3. 生成示例数据预览（可选）
如果用户需要更直观的感受，可生成一个简单的 HTML 表格，展示模拟数据列表。

例如：

| 岗位名称 | 工作城市 | 缺口数 | 状态 |
|---------|---------|-------|------|
| Java工程师 | 上海 | 3 | 招聘中 |
| 产品经理 | 北京 | 2 | 招聘中 |

### 4. 用户确认与反馈
向用户提问：

```markdown
请确认以上信息是否完整、正确？
- 如果正确，请回复"确认"。
- 如果有遗漏，请描述（例如："还需要记录岗位的薪资范围"）。
- 如果有多余，请指出（例如："不需要记录投递时间"）。
```
等待用户回复。

如果用户提出修改，根据描述更新 .ai/tmp/data-model-draft.json，并重新生成业务语言描述（循环）。

如果用户确认无误，保存最终数据模型到 data-model.json。

## Assert
- 每个实体至少有一条业务描述。
- 用户已确认信息完整。

## Artefact
- 更新后的 .ai/tmp/data-model-draft.json（如有修改）
- data-model.json（最终版）

## Analysis
- 如果用户连续两次修改后仍不满意，提示"是否需要查看常见字段模板？"
- 如果用户描述的业务信息与数据模型差异较大，AI 可主动提问："您是否还需要管理【实体】的信息？"

## Adjust
- 如果用户无法描述遗漏字段，AI 可根据常见业务场景推荐（如"是否需要记录岗位的薪资范围？"）。
- 如果用户确认后仍发现遗漏，可通过迭代优化流程（E9.5）补充。

## Advance
- 更新 .ai/进度状态.json，current_step = "B2"。
- 提示用户：下一步将收集业务规则（如数据校验、流程约束）。

## 干预
- 用户必须确认数据模型或提出修改。
