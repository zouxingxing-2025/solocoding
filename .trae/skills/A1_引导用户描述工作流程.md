# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A1`。
- 阶段0所有步骤已完成。

## 目标
通过引导式提问，让用户用自然语言描述业务流程、核心操作和期望结果。

## Action
- 向用户提问："请描述您希望系统支持的核心业务流程。例如：招聘人员如何发布岗位？如何收集候选人？最终需要查看哪些报表？"
- 记录用户的回答，存入 `.ai/进度状态.json` 的 `workflow_desc` 字段。
- 如果用户回答过于简单或模糊，追问："您能举个例子吗？比如第一步做什么，第二步做什么？"

## Assert
- 用户回答至少包含一个业务实体（如"岗位"、"候选人"）。

## Artefact
- 更新 `.ai/进度状态.json`，添加 `workflow_desc` 字段。

## Analysis
- 从用户回答中提取可能的关键词（实体、动作），为后续步骤（A4 提取实体）做准备。

## Adjust
- 若用户无法描述，提供常见领域模板（电商、HR、CRM）供选择。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A2",
    "steps_completed": ["...", "A1"]
  }
  ```
- 提示用户：下一步应执行 A2。

## 干预
用户需回答工作流程描述。
