---
name: 归档经验
description: 将项目临时经验归档到全局经验库，并清理本地副本。
allowed-tools: Read, Write
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `G2`。
- `G1` 已完成。

## 目标
将项目过程中积累的临时经验（位于 `.ai/经验库/待整理/` 和 `.ai/经验库/已分类/`）归档到全局经验库，并清理本地临时副本。

## Action
- 扫描 `.ai/经验库/待整理/` 和 `.ai/经验库/已分类/` 下的所有经验条目。
- 对每条经验：
  - 检查是否已存在于全局经验库（通过标题和内容哈希）。
  - 若不存在，复制到全局经验库的对应分类目录（`~/.ai_experience/...`）。
  - 若存在，合并验证次数和项目来源信息。
- 更新全局经验库索引（`~/.ai_experience/索引.json`）。
- 归档完成后，删除 `.ai/经验库/待整理/` 和 `.ai/经验库/已分类/` 中的本地副本。
- 生成归档报告（`.ai/reports/experience_archive.md`），列出归档的经验条目和数量。

## Assert
- 所有临时经验已归档或合并。
- 本地临时经验目录已清空。

## Artefact
- 归档报告。
- 更新后的全局经验库。

## Analysis
- 若归档过程中发现敏感信息（如密钥），自动脱敏或拒绝归档。
- 记录归档耗时和成功率，用于优化经验库性能。

## Adjust
- 如果全局经验库目录不可写，将经验打包导出为 `experience_export.zip`，供用户手动导入。
- 若用户选择不归档（如项目为临时测试），可跳过此步骤。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "G3",
    "steps_completed": ["...", "G2"]
  }
  ```
- 提示用户：下一步应执行 G3。

## 干预
若需要用户确认是否覆盖已有经验，提示用户选择。
