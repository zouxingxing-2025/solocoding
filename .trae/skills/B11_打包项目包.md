# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B11`。
- `B10` 已完成。

## 目标
将所有生成的模块代码、配置文件、文档等打包成一个完整的项目包，准备进行测试和部署。

## Action
- 收集项目根目录下的所有必要文件：
  - 源代码（`backend/`, `frontend/`, `modules/` 等）
  - 配置文件（`AGENTS.md`, `data-model.json`, `.ai/config/`, `package.json`, `requirements.txt` 等）
  - 文档（`README.md`, `docs/` 等）
  - CI/CD 配置文件
- 排除临时文件和不应交付的目录（`.ai/tmp/`, `.ai/cache/`, `node_modules/`, `__pycache__/` 等）。
- 创建项目包压缩文件（`.zip` 或 `.tar.gz`），命名为 `project-package-{timestamp}.zip`。
- 将压缩包存放到 `.ai/artifacts/` 目录，并记录路径。

## Assert
- 压缩包大小合理（不应包含不必要的大文件）。
- 项目包可在新环境中解压并运行（基本结构完整）。

## Artefact
- 项目包文件（`.ai/artifacts/project-package-*.zip`）

## Analysis
- 检查是否遗漏关键文件（如数据库迁移脚本）。
- 若有多个模块，确保模块间依赖的库已正确声明在根目录的依赖文件中。

## Adjust
- 如果打包失败（如磁盘空间不足），提示用户手动打包。
- 若检测到不应包含的文件（如 `.env` 文件），自动排除。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B12",
    "steps_completed": ["...", "B11"]
  }
  ```
- 提示用户：下一步应执行 B12。

## 干预
无。
