# 运行沙箱测试

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E3`。
- `E2` 已完成（测试用例已生成）。

## 目标
在沙箱环境中运行所有测试用例（单元测试、集成测试），收集测试结果（通过/失败）和代码覆盖率，特别关注业务规则验证。

## Action
- 进入沙箱环境（复用 C1 的环境）。
- 执行测试命令：
  - Python: `pytest tests/ --cov=src --cov-report=html`
  - Node.js: `npm test -- --coverage`
- 捕获输出，记录总用例数、通过数、失败数、错误数、跳过数。
- 生成测试报告（`.ai/reports/test_report.html`）和覆盖率报告。
- 若测试失败，将失败用例的详细信息记录到 `.ai/reports/failed_tests.log`。

## Assert
- 测试执行完成，无异常中断。
- 核心功能模块的测试通过率不低于 90%（可配置）。
- 所有业务规则对应的测试用例必须通过（Critical）。

## Artefact
- 测试报告（`.ai/reports/test_report.html`）
- 失败日志（`.ai/reports/failed_tests.log`）

## Analysis
- 分析失败原因：断言错误、环境问题、代码 bug、业务规则未正确实现。
- 若覆盖率低于阈值，标记需要补充测试的模块。

## Adjust
- 如果测试失败且原因是环境问题，自动修复环境后重试。
- 若失败用例过多（>10%），提示用户优先修复。
- 如果业务规则测试失败，AI 应尝试分析根本原因（如数据模型未正确约束）。

## Advance
- 更新 .ai/进度状态.json：
  ```json
  {
    "current_step": "E4",
    "steps_completed": ["...", "E3"]
  }
  ```
- 提示用户：下一步应执行 E4。

## 干预
- 无。
