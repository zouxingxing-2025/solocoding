# 提取关键概念

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A1.5`。
- `A1` 已完成（用户已描述工作流程）。

## 目标
从用户描述的工作流程中提取领域关键词、核心概念、隐性知识，生成术语表 `docs/术语表.md`，供后续步骤参考。

## Action
1. 读取 `.ai/进度状态.json` 中的 `workflow_desc` 字段。
2. 识别名词和专有术语（如"岗位"、"候选人"、"缺口数"、"转化率"等）。
3. 对每个术语提供简要定义（如果用户未明确定义，AI 根据上下文推断）。
4. 提取可能的隐性知识（如"缺口数 = 需求人数 - 已招人数"）。
5. 将术语表写入 `docs/术语表.md`，格式为 Markdown 表格：

   | 术语 | 定义 | 关联概念 |
   |------|------|----------|
   | 岗位 | 招聘职位 | 候选人、缺口数 |
   | ... | ... | ... |

6. 更新进度状态，标记 `terms_extracted: true`。

## Assert
- 术语表至少包含3个关键术语。
- 定义清晰，无歧义。

## Artefact
- `docs/术语表.md`

## Analysis
- 如果术语过多（>20），考虑按类别分组。
- 若某些术语定义模糊，AI 可向用户提问确认。

## Adjust
- 如果用户未提供足够信息，可跳过此步骤（标记 `terms_extracted: false`）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A2",
    "steps_completed": ["A1", "A1.5"]
  }
  ```
  提示用户：下一步应执行 A2。

When to Use This Skill
在需求分析阶段，用户描述完工作流程后。

当项目涉及专业领域术语时。

Not For
用户已经提供了明确术语定义的情况。

非文本描述的需求（如图表、原型图）。

## When to Use This Skill
Use this skill when A1.5_提取关键概念 is required, such as during phase A tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
