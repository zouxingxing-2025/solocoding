# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E8`。
- `E7` 已完成（部署更新完成）。

## 目标
将迭代变更后的最新信息（如新增模块、修改的配置、部署版本等）同步到核心契约 `AGENTS.md` 中，确保文档与代码一致。

## Action
- 读取当前 `AGENTS.md` 和 `.ai/进度状态.json` 中的变更记录。
- 更新以下章节：
  - 第6章"功能模块清单"：根据最新 `modules.yaml` 更新模块状态、路径。
  - 第7章"插件配置"：如有插件变更，同步更新。
  - 第8章"部署配置"：记录新的部署版本和时间戳。
- 若变更涉及数据模型，更新第3章的数据边界引用（但数据模型本身已独立更新，仅检查引用一致性）。
- 保存更新后的 `AGENTS.md`。

## Assert
- 变更内容已正确反映在核心契约中。
- 无残留占位符。

## Artefact
- 更新后的 `AGENTS.md`。

## Analysis
- 检查是否有其他章节（如第9章生命周期）需要同步更新（如模块状态变化）。
- 验证文档与代码的一致性（可触发 B9.1 检查）。

## Adjust
- 若发现不一致，自动修正或提示用户。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E9.1",
    "steps_completed": ["...", "E8"]
  }
  ```
- 提示用户：下一步应执行 E9.1（缺陷修复工作流的第一步）。

## 干预
无。


## When to Use This Skill
Use this skill when E8_更新核心契约 is required, such as during phase E tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
