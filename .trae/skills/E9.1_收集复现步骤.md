# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E9.1`。
- `E8` 已完成。

## 目标
收集用户报告的缺陷的详细复现步骤，包括操作路径、输入数据、期望结果和实际结果。这是缺陷修复流程的第一步，必须完成复现才能进入后续修复。

## Action
- 向用户提问："请描述您遇到的问题。请提供以下信息：1）在哪个页面或接口？2）执行了什么操作？（例如点击按钮、提交表单、调用API）3）输入了什么数据？4）期望看到什么结果？5）实际看到了什么结果？"
- 如果用户提供的信息不完整，追问具体细节（如"您能一步步告诉我如何复现吗？"）。
- 将用户回答结构化存储到 `.ai/缺陷报告/当前缺陷/复现步骤.md`。
- 尝试在本地或沙箱中按照步骤复现（若可能），确认缺陷存在。

## Assert
- 用户已提供足够的复现信息（至少包含操作步骤和实际结果）。
- 若无法复现，标记为"待补充信息"。

## Artefact
- 复现步骤文档（`.ai/缺陷报告/当前缺陷/复现步骤.md`）。

## Analysis
- 分析复现步骤是否清晰、无歧义。
- 若用户多次提供模糊信息，记录为沟通经验。

## Adjust
- 如果无法复现，返回提问并请求用户提供更多细节（如截图、日志、环境信息）。
- 若用户明确表示无法提供更多，标记为"无法复现"，结束流程。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E9.2",
    "steps_completed": ["...", "E9.1"]
  }
  ```
- 提示用户：下一步应执行 E9.2（在沙箱中复现缺陷）。

## 干预
用户需提供复现步骤描述。


## When to Use This Skill
Use this skill when E9.1_收集复现步骤 is required, such as during phase E tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
