# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B2`。
- `B1` 已完成。

## 目标
根据项目领域（如电商、HR、CRM）加载对应的功能模板库（`function-templates.yaml`），供后续匹配候选功能使用。

## Action
- 读取 `.ai/进度状态.json` 中的 `domain` 字段（从用户描述中推断或用户确认）。
- 根据领域，从全局经验库或内置模板库中加载对应的 `function-templates.yaml` 文件。
- 如果领域模板不存在，加载通用模板（`general-templates.yaml`）。
- 将模板内容缓存到 `.ai/tmp/function-templates-filtered.yaml`。

## Assert
- 模板文件存在且格式正确。
- 至少有一个功能模板。

## Artefact
- `.ai/tmp/function-templates-filtered.yaml`

## Analysis
- 检查模板中的字段语义是否与数据模型匹配。
- 若模板过时或不适配，提示用户更新。

## Adjust
- 如果无匹配模板，允许用户自定义功能（后续 B5）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B3",
    "steps_completed": ["...", "B2"]
  }
  ```
- 提示用户：下一步应执行 B3。

## 干预
无。
