# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A4`。
- `A3` 已完成。

## 目标
从用户描述的工作流程和页面需求中提取核心业务实体。

## Action
- 读取进度状态中的 `workflow_desc`、`input_pages`、`output_pages`。
- 识别名词（如"岗位"、"候选人"、"订单"），去重。
- 输出实体列表，存入进度状态的 `entities`。

## Assert
- 至少有一个实体。

## Artefact
- 进度状态中的 `entities` 列表。

## Analysis
- 若实体过多（>10），考虑是否拆分或合并。

## Adjust
- 如果实体不明确，向用户提问："您提到的XX是否应该作为一个独立实体？"

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A5",
    "steps_completed": ["...", "A4"]
  }
  ```
- 提示用户：下一步应执行 A5。

## 干预
无。


## When to Use This Skill
Use this skill when A4_提取实体 is required, such as during phase A tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
