# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `C2`。
- `C1` 已完成。

## 目标
根据功能模块的需求文档和验收标准，自动生成功能测试用例（单元测试、集成测试）。

## Action
- 读取每个模块的 `specs/features/*/需求.md` 和 `技术方案.md`，提取验收标准。
- 对于每个验收标准，生成对应的测试用例：
  - 输入页面：测试字段校验、提交动作、错误处理。
  - 展示页面：测试数据展示、聚合计算、筛选功能。
  - API 接口：测试请求/响应格式、状态码、业务逻辑。
- 使用测试框架（如 pytest、Jest）编写测试脚本。
- 将测试文件写入 `tests/` 目录（按模块组织）。

## Assert
- 每个功能模块至少有一个测试用例。
- 测试用例覆盖所有验收标准。

## Artefact
- 测试脚本文件（`tests/` 目录下）。

## Analysis
- 检查是否有重复测试用例，合并相似逻辑。
- 若某个验收标准无法自动测试（如 UI 动画），标记为"需手动测试"。

## Adjust
- 如果生成测试用例时发现需求不明确，提示用户补充验收标准。
- 若测试框架未配置，自动生成配置文件（如 `pytest.ini`、`jest.config.js`）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "C3",
    "steps_completed": ["...", "C2"]
  }
  ```
- 提示用户：下一步应执行 C3。

## 干预
无。
