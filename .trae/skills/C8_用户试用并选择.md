# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `C8`。
- `C7.5` 已完成（集成测试通过或已知问题已标记）。

## 目标
用户试用多个候选demo，选择最满意的一个（人工干预点3）。AI 等待用户选择，并记录选择结果。

## Action
- 读取 `.ai/demos/preview_urls.json`，获取每个demo的访问URL。
- 向用户展示demo列表及简要说明（如"Demo A: 使用 Ant Design，功能完整；Demo B: 使用 Material-UI，加载更快"）。
- 提问："请试用以上demo，并选择您最满意的一个（输入A、B或C）。如果不满意，可要求重新生成。"
- 等待用户回复。
- 记录用户选择的demo ID到 `.ai/进度状态.json` 的 `selected_demo` 字段。

## Assert
- 用户已做出选择或要求重新生成。

## Artefact
- 更新进度状态中的 `selected_demo`。

## Analysis
- 若用户要求重新生成，返回 C6 调整生成策略。
- 记录用户偏好（如"喜欢轻量级UI"），作为成功经验存入全局经验库。

## Adjust
- 如果用户未回复超时（如10分钟），默认选择第一个demo。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "C9",
    "steps_completed": ["...", "C8"]
  }
  ```
- 提示用户：下一步应执行 C9。

## 干预
用户需选择demo或要求重新生成。


## When to Use This Skill
Use this skill when C8_用户试用并选择 is required, such as during phase C tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
