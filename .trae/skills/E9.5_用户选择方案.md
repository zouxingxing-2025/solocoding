# E9.5_用户选择方案.md

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E9.5`。
- `E9.4` 已完成（修复方案已生成）。

## 目标
向用户展示多个修复方案，由用户选择一个执行。若用户未选择，默认使用推荐方案。

## Action
- 读取 `.ai/缺陷报告/当前缺陷/修复方案.md` 中的方案列表。
- 向用户展示每个方案的摘要（修改范围、复杂度、风险）。
- 提问："请选择您希望使用的修复方案（输入方案编号，如 1、2 等）。如果不选择，将默认使用推荐方案。"
- 等待用户回复。
- 若用户指定方案，记录到进度状态；若用户未指定或超时（如 60 秒），自动选择推荐方案。

## Assert
- 用户已做出选择（或超时后自动选择）。
- 所选方案在方案列表中存在。

## Artefact
- 更新进度状态中的 `selected_fix_plan` 字段。

## Analysis
- 如果用户选择非推荐方案，记录为偏好作为经验。
- 若用户提出修改建议，返回 E9.4 调整方案。

## Adjust
- 若用户回复不明确，再次展示选项并要求明确选择。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E9.6",
    "steps_completed": ["...", "E9.5"]
  }
  ```
- 提示用户：下一步应执行 E9.6（执行修复）。

## 干预
用户需选择方案或确认默认选择。
