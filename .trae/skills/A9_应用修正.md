# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A9`。
- `A8` 已完成且用户提出了修改意见。

## 目标
根据用户在 A8 中提出的修改指令，更新数据模型草稿。

## Action
- 读取用户反馈（`.ai/进度状态.json` 中的 `user_feedback`）。
- 解析每条修改指令（例如："给实体 Position 添加字段 salary，类型 float"）。
- 逐条应用到 `.ai/tmp/data-model-draft.json`。
- 记录修改前后的差异。

## Assert
- 所有指令都已解析并成功应用。
- 修改后的数据模型仍通过 schema 验证。

## Artefact
- 更新后的 `.ai/tmp/data-model-draft.json`。
- 差异日志（可选，存于 `.ai/logs/`）。

## Analysis
- 检查修改是否引入新的不一致（例如，删除某字段后关系中外键引用失效）。
- 如果修改导致关系断裂，提示用户并提供修复建议。

## Adjust
- 如果某条指令无法执行（如删除不存在的字段），向用户反馈并请求澄清。
- 如果用户一次性修改过多，可分批应用并逐条确认。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A10",
    "steps_completed": ["...", "A9"]
  }
  ```
- 提示用户：下一步应执行 A10。

## 干预
如果指令模糊或无法执行，AI 需再次请求用户澄清。


## When to Use This Skill
Use this skill when A9_应用修正 is required, such as during phase A tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
