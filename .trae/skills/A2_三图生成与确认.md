# 三图生成与确认

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 为 `A2`。
- 用户已完成初步工作流程描述（存储在 `workflow_desc` 中）。

## 目标
生成三种图表（业务流程图、泳道图、权限矩阵），帮助用户可视化确认业务流程、角色职责和权限边界。用户通过自然语言步骤列表修改，AI 同步更新图表。最终生成结构化的角色、任务、字段、权限数据。

## Action

### 1. 准备 MCP 服务
- 检查 MCP 服务 `drawio` 是否可用（TRAE 环境应已配置）。
- 如果不可用，提示用户安装 `@next-ai-drawio/mcp-server`，并回退到自然语言步骤列表编辑模式。

### 2. 生成业务流程图
- 调用 MCP 工具 `drawio` 的 `generate_diagram` 功能，传入用户描述：
```
mcp_call_tool("drawio", "generate_diagram", {
"prompt": "根据以下业务流程描述，生成业务流程图（Mermaid格式），展示任务节点和流转方向，不区分角色：\n{workflow_desc}",
"format": "mermaid"
})
```
- 获取返回的 Mermaid 代码。
- 在 TRAE 预览窗口中渲染流程图。

### 3. 生成泳道图
- 再次调用 MCP 工具，要求生成泳道图：
```
mcp_call_tool("drawio", "generate_diagram", {
"prompt": "根据以下业务流程描述，生成泳道图（Mermaid格式），区分角色（泳道）。每个任务节点内标注涉及的数据字段（如"发布岗位：岗位名称、城市、缺口数"）。角色从描述中提取（如HR、候选人、经理）。\n{workflow_desc}",
"format": "mermaid"
})
```
- 获取返回的 Mermaid 代码。
- 在 TRAE 预览窗口中渲染泳道图。

### 4. 生成权限矩阵
- 基于泳道图中的角色和任务，生成权限矩阵表（Markdown 格式）：

| 角色 | 岗位表（读） | 岗位表（写） | 候选人表（读） | 候选人表（写） | 审批表（读） | 审批表（写） |
|------|------------|------------|-------------|-------------|------------|------------|
| HR | ✅ | ✅ | ✅ | ❌ | ✅ | ❌ |
| 经理 | ✅ | ❌ | ✅ | ✅（仅审批） | ✅ | ✅ |
| 候选人 | ✅ | ❌ | ✅（仅自己） | ✅（仅自己） | ❌ | ❌ |

- 注意：权限矩阵的初始内容由 AI 根据常见业务模式推断，用户后续可修改。

### 5. 展示可编辑的步骤列表
- 在图表下方，生成一个自然语言步骤列表（Markdown 格式），用户可直接编辑：
```markdown
## 业务流程步骤（可编辑）
1. HR 发布岗位（字段：岗位名称、城市、缺口数）
2. 候选人投递简历（字段：姓名、手机号、应聘岗位）
3. 经理审批（字段：申请ID、状态、意见）
```
用户修改列表后，点击"重新生成图表"，AI 重新调用 MCP 服务更新所有图表。

### 6. 用户确认
向用户提问：

```markdown
请确认以上流程图、泳道图和权限矩阵是否完整、正确。
- 如果正确，请回复"确认"。
- 如果需要修改，请直接编辑步骤列表，或描述修改内容。
```
等待用户回复。

如果用户要求修改，重复步骤 2-5。

如果用户确认，提取步骤列表中的结构化数据（角色、任务、字段、权限），保存到 .ai/temp/roles_tasks_fields.json 和 .ai/temp/permission_matrix.json。

## Assert
- 每个角色至少有一个任务。
- 每个任务至少有一个字段。
- 权限矩阵中每个角色对每个实体的权限已定义（默认只读或空）。

## Artefact
- .ai/temp/workflow.mmd（业务流程图）
- .ai/temp/swimlane.mmd（泳道图）
- .ai/temp/permission_matrix.json（权限矩阵）
- .ai/temp/roles_tasks_fields.json（角色任务字段）

## Analysis
- 如果 MCP 服务不可用，回退到自然语言步骤列表编辑模式（不生成图表），并提示用户手动安装 MCP 服务以获得更好体验。
- 如果用户连续两次修改后仍不满意，提示"是否需要帮助？可提供更详细的业务描述。"

## Adjust
- 如果用户对权限矩阵不满意，可提供"权限模板"供选择（如"HR 全部权限"、"经理只读"）。
- 如果用户无法描述字段，AI 可根据常见业务模式推荐字段。

## Advance
- 更新 .ai/进度状态.json，current_step = "A3"。
- 提示用户：下一步将进行需求饱和确认。

## 干预
- 用户必须确认图表或提出修改。

- 用户通过可编辑的步骤列表修改，无需直接编写 Mermaid 代码。
