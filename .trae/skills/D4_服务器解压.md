# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D4`。
- `D3` 已完成（制品已上传）。

## 目标
在目标服务器上解压上传的制品，准备运行环境。

## Action
- 通过 SSH 连接到服务器（使用已配置的认证方式）。
- 切换到制品存放目录（如 `/opt/app/`）。
- 根据制品类型执行解压命令：
  - 如果是 `.tar.gz`：`tar -xzf project-name-*.tar.gz`
  - 如果是 `.zip`：`unzip -o project-name-*.zip`
- 解压后，检查关键文件是否存在（如 `docker-compose.yml`、`start.sh`、`backend/` 目录等）。
- 记录解压输出和错误日志。

## Assert
- 解压成功，无错误。
- 关键文件存在。

## Artefact
- 解压后的代码目录（如 `/opt/app/project-name/`）。
- 解压日志（`.ai/logs/extract.log`）。

## Analysis
- 如果解压失败，检查磁盘空间、文件完整性、权限问题。
- 若存在同名目录，询问用户是否覆盖（默认备份旧目录到 `.backup`）。

## Adjust
- 如果目标目录已存在且用户未确认覆盖，则中止步骤并提示用户。
- 若缺少关键文件，输出错误报告并回退到 D3。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D5",
    "steps_completed": ["...", "D4"]
  }
  ```
- 提示用户：下一步应执行 D5。

## 干预
若需要覆盖现有目录，需用户确认。
