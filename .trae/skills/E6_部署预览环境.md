# 部署预览环境

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E6`。
- `E5` 已完成（对抗性审查通过）。

## 目标
为最终确认的界面原型部署静态预览环境（使用 TRAE 沙盒内嵌浏览器），无需启动真实服务器，降低本地开发成本。

## Action
1. 读取最终确认的界面原型（`.ai/previews/app_preview.html`）。
2. 确保原型包含完整的静态文件（HTML/CSS/JS）。
3. 使用 TRAE 的"预览"工具直接打开 `app_preview.html`，无需启动 HTTP 服务器。
4. 记录预览 URL（格式：`file:///.ai/previews/app_preview.html`）。
5. 向用户输出：
   ```markdown
   ✅ 预览环境已就绪：
   - [点击预览](file:///.ai/previews/app_preview.html)

   请试用核心功能：
   - 创建一条记录
   - 编辑、删除
   - 筛选、搜索
   - 验证业务规则（如手机号格式）

   试用后请回复"确认"或提出修改意见。
   ```

## Assert
- 预览文件存在且可访问。
- 用户可正常交互。

## Artefact
- 预览 URL 记录（.ai/previews/preview_url.txt）

## Analysis
- 如果预览文件缺失，提示用户重新运行 C1 生成原型。

## Adjust
- 若用户提出修改意见，返回 C1 调整界面原型。

## Advance
- 更新 .ai/进度状态.json：
  ```json
  {
    "current_step": "E7",
    "steps_completed": ["...", "E6"]
  }
  ```
- 提示用户：下一步应执行 E7（集成测试）。

## 干预
- 用户需确认预览效果。
