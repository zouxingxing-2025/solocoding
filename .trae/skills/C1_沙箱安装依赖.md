# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `C1`。
- `B13` 已完成。

## 目标
在沙箱环境中安装项目所需的所有依赖（如 Python 包、Node.js 模块），确保安装过程可复现且不污染主机环境。

## Action
- 读取项目根目录下的依赖文件（`requirements.txt`、`package.json` 等）。
- 创建临时虚拟环境或容器（使用 Docker 或 Python venv）。
- 执行依赖安装命令：
  - Python: `pip install -r requirements.txt`
  - Node.js: `npm install`
- 记录安装输出和错误日志。
- 若安装失败，分析错误原因（网络、版本冲突、缺少系统库）。

## Assert
- 所有依赖安装成功，无报错。
- 关键模块（如 Flask、React）可正常导入。

## Artefact
- 安装日志（`.ai/logs/deps_install.log`）。

## Analysis
- 检查是否有版本冲突，若存在则尝试自动降级或升级。
- 记录安装耗时，用于后续优化。

## Adjust
- 如果依赖安装失败，尝试更换镜像源（如 pip 使用清华源）后重试，最多3次。
- 若仍失败，提示用户手动安装并跳过。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "C2",
    "steps_completed": ["...", "C1"]
  }
  ```
- 提示用户：下一步应执行 C2。
