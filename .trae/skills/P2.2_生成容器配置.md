# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `P2.2`。
- `P2.1` 已完成（无论成功或跳过）。

## 目标
创建或修改项目根目录下的 `.devcontainer/devcontainer.json` 文件，添加 SSH Agent 转发配置。

## Action
- 检查 `.devcontainer/devcontainer.json` 是否存在。
- 如果不存在，创建默认配置（使用基础镜像 `mcr.microsoft.com/devcontainers/base:ubuntu`）。
- 读取现有配置（如果存在）。
- 合并以下配置项（避免重复）：
  ```json
  "mounts": [
    "source=${localEnv:SSH_AUTH_SOCK},target=/ssh-agent,type=bind"
  ],
  "remoteEnv": {
    "SSH_AUTH_SOCK": "/ssh-agent"
  },
  "postCreateCommand": "git config --global --add --bool push.autoSetupRemote true"
  ```
- 保存文件。

## Assert
- 文件路径正确（`.devcontainer/devcontainer.json`）。
- JSON 格式有效。
- `mounts` 和 `remoteEnv` 字段已正确添加。

## Artefact
- 更新后的 `.devcontainer/devcontainer.json`。

## Analysis
- 检查原有配置是否已有 `mounts` 或 `remoteEnv`，避免重复或覆盖。
- 如果主机 SSH Agent socket 路径不是标准位置（如 macOS 上的 `/run/host-services/ssh-auth.sock`），需调整 `source` 路径（但标准 `${localEnv:SSH_AUTH_SOCK}` 通常可用）。

## Adjust
- 若用户使用自定义镜像，确保镜像中已安装 `openssh-client`（否则在 `postCreateCommand` 中安装）。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "P2.3",
    "steps_completed": ["P1", "P2", "P2.1", "P2.2"]
  }
  ```
- 提示用户：下一步应执行 P2.3。

无（自动生成，用户可后续手动调整）。
