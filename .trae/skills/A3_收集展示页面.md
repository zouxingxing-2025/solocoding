# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A3`。
- `A2` 已完成。

## 目标
明确用户需要哪些展示页面（报表、列表），每个页面包含哪些字段、聚合方式、筛选条件。

## Action
- 向用户提问："请列出您需要的展示页面（例如：岗位缺口汇总表、候选人列表）。对于每个页面，告诉我：1）需要展示哪些字段？2）是否需要聚合（如按城市分组、求和）？3）支持哪些筛选条件（如日期范围、状态）？"
- 将用户回答结构化为 JSON 格式，存入 `.ai/进度状态.json` 的 `output_pages` 字段。
- 如果用户未提及聚合，默认按明细展示。

## Assert
- 每个展示页面至少有字段列表。

## Artefact
- 进度状态中的 `output_pages` 字段。

## Analysis
- 识别聚合需求（SUM, COUNT, GROUP BY），为后续数据转换路径做准备。

## Adjust
- 如果用户描述模糊，提供常见报表模板（如"销售汇总表"、"用户列表"）供参考。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "A4",
    "steps_completed": ["...", "A3"]
  }
  ```
- 提示用户：下一步应执行 A4。

## 干预
用户需描述展示页面。


## When to Use This Skill
Use this skill when A3_收集展示页面 is required, such as during phase A tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
