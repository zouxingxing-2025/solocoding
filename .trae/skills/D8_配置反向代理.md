# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D8`。
- `D7` 已完成（应用服务已启动）。

## 目标
配置反向代理（如 Nginx、Apache）将外部请求转发到应用服务，并处理静态文件、负载均衡、SSL 终止等。

## Action
- 读取 `.ai/config/deployment.yaml` 中的域名、端口、SSL 配置。
- 生成反向代理配置文件（如 `nginx.conf` 片段）：
  - 监听 80 端口（HTTP）和 443 端口（HTTPS，若启用 SSL）。
  - 将请求代理到本地应用服务地址（如 `http://localhost:8000`）。
  - 配置静态文件路径（如 `/static/`、`/media/`）。
  - 设置请求头（Host、X-Real-IP、X-Forwarded-For）。
- 将配置文件上传到服务器的 `/etc/nginx/sites-available/` 目录。
- 创建符号链接到 `/etc/nginx/sites-enabled/`。
- 测试配置语法（`nginx -t`）。
- 重载 Nginx（`systemctl reload nginx` 或 `nginx -s reload`）。

## Assert
- 配置文件语法正确。
- 重载后 Nginx 运行正常。

## Artefact
- 生成的配置文件（`.ai/config/nginx.conf`）。
- 部署日志（`.ai/logs/nginx_setup.log`）。

## Analysis
- 检查端口是否被占用，若冲突则提示用户修改。
- 若 SSL 证书尚未配置，跳过 HTTPS 部分，后续由 D9 处理。

## Adjust
- 如果 Nginx 未安装，提示用户安装（提供命令）。
- 若配置文件测试失败，自动修正常见错误（如缺少分号、路径错误）。

## Advance

  ```json
  {
    "current_step": "D9",
    "steps_completed": ["...", "D8"]
  }
  ```
- 提示用户：下一步应执行 D9。
