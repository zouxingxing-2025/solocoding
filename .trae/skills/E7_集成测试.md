# 集成测试

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E7`。
- `E6` 已完成（预览环境部署）。

## 目标
测试前后端集成、API 调用、业务规则执行和权限控制，确保系统各组件协同工作。

## Action
- 启动后端服务（如 `npm run dev`）。
- 执行集成测试脚本：
  - 测试 API 调用（创建、读取、更新、删除）。
  - 测试业务规则执行（如手机号格式校验、唯一性约束）。
  - 测试权限控制（不同角色访问不同接口）。
- 记录测试结果，生成集成测试报告（`.ai/reports/integration_test.md`）。

## Assert
- 所有集成测试通过。
- 业务规则在前后端都得到执行。

## Artefact
- 集成测试报告（`.ai/reports/integration_test.md`）。

## Analysis
- 分析集成失败原因，如 API 格式不匹配、权限配置错误。

## Adjust
- 若集成测试失败，修复代码后重试。

## Advance
- 更新 .ai/进度状态.json：
  ```json
  {
    "current_step": "F1",
    "steps_completed": ["...", "E7"]
  }
  ```
- 提示用户：下一步应执行 F1（生成最终交付物清单）。

## 干预
- 无。
