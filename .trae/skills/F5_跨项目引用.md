name: 跨项目引用
description: 跨项目引用全局经验库，在新项目中复用已有经验。
allowed-tools: Read
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F5`。
- `F4` 已完成（全局规则已提升）。

## 目标
在新项目初始化时，加载全局经验库索引，供后续步骤按需查询和引用，实现跨项目经验复用。

## Action
- 读取全局经验库路径（`~/.ai_experience/`）下的 `index.json`。
- 将索引摘要缓存到当前项目的 `.ai/全局经验库缓存.json`（仅索引，不加载具体内容）。
- 如果全局经验库中存在与当前项目技术栈、领域匹配的全局规则（`global-rules/common-constraints.yaml`），将其合并到 `AGENTS.md` 模板的第0章（作为默认约束）。
- 记录引用的经验库版本号到 `.ai/进度状态.json`。

## Assert
- 索引缓存已创建。
- 若存在全局规则，已合并到核心契约模板。

## Artefact
- `.ai/全局经验库缓存.json`
- 更新后的 `AGENTS.md` 模板（可选）。

## Analysis
- 检查全局规则是否与当前项目技术栈冲突（例如规则要求 Python 3.10+，但项目使用 3.9）。
- 若冲突，提示用户手动调整。

## Adjust
- 如果全局经验库不可访问，跳过引用并记录警告。
- 若用户选择不使用全局经验库，可设置 `use_global_experience: false`。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "G1",
    "steps_completed": ["...", "F5"]
  }
  ```
- 提示用户：阶段6完成，下一步进入阶段7（清理与交付），应执行 G1。

## 干预
无。
