# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F1`。
- 阶段5已完成（或项目进行中）。

## 目标
在项目执行过程中，自动识别可记录的经验场景（如用户修正、测试失败、模块高分等），并生成结构化经验条目，存入待整理目录。

## Action
- 监控以下触发点（贯穿所有阶段）：
  - 用户修正AI生成的内容（如A9、B5）。
  - 用户从多个demo中选择一个（C8）。
  - 沙箱测试失败（C3、C4）。
  - 对抗性审查发现问题（A14、B13、C5、D12）。
  - 模块预评估得分显著高于其他候选（B3）。
  - 同一错误重复发生（如依赖冲突多次出现）。
- 当触发条件满足时，自动生成经验条目：
  - 成功经验：记录成功做法、用户偏好、高评分模块。
  - 失败教训：记录错误现象、原因、解决方案、相关技术栈。
- 将经验条目写入 `.ai/经验库/待整理/YYYY-MM-DD_类型_简短描述.md`，格式遵循核心契约第14章模板。
- 更新 `.ai/经验库/待整理/索引.json`（可选）。

## Assert
- 经验条目已生成，内容完整。
- 触发点未被遗漏（若发现遗漏，记录为改进点）。

## Artefact
- 经验条目文件（待整理目录下）。

## Analysis
- 分析经验是否具有通用性，标记可能提升为全局规则的候选。
- 若同一类型的经验频繁出现，提示用户可能需要优化流程。

## Adjust
- 如果用户关闭自动记录功能，跳过触发点监控。
- 若敏感信息（如密钥）出现在日志中，自动脱敏后存储。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "F2",
    "steps_completed": ["...", "F1"]
  }
  ```
- 提示用户：下一步应执行 F2（经验存储）。

## 干预
无（全自动）。
