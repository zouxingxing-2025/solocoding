# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `P2`。
- `P1` 已完成。

## 目标
配置 Dev Container 支持 SSH Agent 转发，使沙箱环境能复用主机的 SSH 密钥。

## Action
- 检测主机操作系统（macOS / Linux / Windows）。
- 检查主机 SSH Agent 是否运行（`ssh-add -l`）。
- 如果未运行，提示用户启动 SSH Agent 并添加私钥。
- 在项目根目录创建/修改 `.devcontainer/devcontainer.json`，添加以下配置：
  ```json
  "mounts": [
    "source=${localEnv:SSH_AUTH_SOCK},target=/ssh-agent,type=bind"
  ],
  "remoteEnv": {
    "SSH_AUTH_SOCK": "/ssh-agent"
  },
  "postCreateCommand": "git config --global --add --bool push.autoSetupRemote true"
  ```
- 保存文件。

## Assert
- 主机 SSH Agent 正在运行且包含至少一个私钥。
- `.devcontainer/devcontainer.json` 文件存在且 JSON 格式有效。

## Artefact
- `.devcontainer/devcontainer.json` 文件。

## Analysis
- 如果主机 SSH Agent 未运行，提供具体启动命令。
- 如果配置失败，检查路径挂载权限。

## Adjust
- 若无法自动配置，提供手动配置指南，并标记 `ssh_agent_ready: false`。
- 若用户跳过，记录警告。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "P2.1",
    "steps_completed": ["P1", "P2"]
  }
  ```
- 提示用户：下一步应执行 P2.1。

## 干预
- 用户需确认 SSH Agent 已启动（如果自动检测失败）。
- 用户需提供私钥路径（如果默认路径不存在）。
