# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `A8`。
- `A7` 已完成。

## 目标
将生成的 `data-model.json` 草稿以可读格式（表格、树形结构）展示给用户，并请求用户确认或提出修改意见。这是第一个关键的人工干预点。

## Action
- 读取 `.ai/tmp/data-model-draft.json`。
- 生成人类可读的摘要：
  - 实体列表（每个实体附上字段表格：名称、类型、必填、主键）。
  - 关系列表（来源 → 目标，关系类型）。
- 向用户提问："请确认以上数据模型是否正确？您可以指出需要修改的地方，例如：添加/删除字段、修改类型、调整关系。"
- 等待用户回复（可以一次提出多个修改）。

## Assert
- 摘要已展示给用户。
- 用户已回复（确认或提出修改）。

## Artefact
- 用户确认或修改记录存入 `.ai/进度状态.json` 的 `user_feedback` 字段。

## Analysis
- 如果用户提出修改，分析修改类型（增/删/改字段、改关系、改类型）。

## Adjust
- 根据用户修改指令，调用 `A9` 更新数据模型草稿。

## Advance
- 若用户确认无误，更新状态：`current_step = "A11"`，跳过 A9、A10，直接进入 A11。
- 若用户提出修改，更新状态：`current_step = "A9"`，进入 A9。

## 干预
- 用户必须确认或提出修改。
