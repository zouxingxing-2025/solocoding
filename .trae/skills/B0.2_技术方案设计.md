# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B0.2`。
- `B0.1` 已完成。

## 目标
基于功能需求文档和全局数据模型，设计详细的技术方案，包括 API 接口定义、数据库表结构变更、核心算法/业务逻辑、异常处理等。

## Action
- 读取当前模块的 `specs/features/*/需求.md` 和全局 `data-model.json`。
- 设计 API 接口：
  - 确定请求方法、路径、参数、响应格式。
  - 使用 OpenAPI 片段描述。
- 设计数据库变更：
  - 新增表、字段、索引、外键约束。
  - 若涉及修改，生成迁移脚本（SQL 或 ORM 迁移）。
- 设计核心逻辑：
  - 主要流程步骤、算法（如计算、校验、数据转换）。
  - 依赖的外部服务或模块。
- 设计异常处理：
  - 错误类型、错误码、用户提示。
- 将技术方案写入 `specs/features/*/技术方案.md`。

## Assert
- 技术方案包含 API、数据库、核心逻辑、异常处理四个部分。
- 所有引用的数据模型字段在 `data-model.json` 中存在。

## Artefact

## Analysis
- 检查方案是否与现有架构冲突（如重复的 API 路径）。
- 评估实现复杂度和风险。

- 若用户要求多种实现方案，生成多个候选。
