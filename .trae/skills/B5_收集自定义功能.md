name: 收集自定义功能
description: 允许用户描述系统未能自动匹配的自定义功能。
allowed-tools: Read, Write
---

# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B5`。
- `B4` 已完成。

## 目标
允许用户描述自动匹配未能覆盖的自定义功能，例如基于字段的简单算术组合或特定业务逻辑。

## Action
- 向用户提问："如果推荐的功能列表中没有您需要的，请描述您想要的自定义功能。例如：'平均订单金额 = 总销售额 / 订单数'。请用'结果名称 = 字段A / 字段B'的格式描述。"
- 解析用户输入，提取：功能名称、表达式、涉及字段、输出类型。
- 验证表达式中引用的字段是否存在于 `data-model.json` 中。
- 将有效的自定义功能添加到候选列表，存入 `.ai/tmp/custom_functions.json`。

## Assert
- 用户提供的表达式至少包含一个字段。
- 所有字段都在数据模型中存在。

## Artefact
- `.ai/tmp/custom_functions.json`

## Analysis
- 检查表达式是否包含非法运算（如除以零、不支持的函数）。
- 如果用户描述不清晰，给出具体示例并再次提问。

## Adjust
- 若字段不存在，提示用户修正字段名或添加字段。
- 若表达式无效，引导用户使用简单四则运算。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B6",
    "steps_completed": ["...", "B5"]
  }
  ```
- 提示用户：下一步应执行 B6。

## 干预
用户需提供自定义功能描述（可跳过）。


## When to Use This Skill
Use this skill when B5_收集自定义功能 is required, such as during phase B tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
