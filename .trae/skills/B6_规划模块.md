# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B6`。
- `B5` 已完成。

## 目标
为每个勾选的功能规划独立的代码模块，确定模块边界、依赖关系、目录结构。

## Action
- 读取 `selected_functions` 和 `custom_functions.json`。
- 为每个功能生成一个模块ID（如 `mod_001`）。
- 确定模块之间的依赖关系（如模块B依赖模块A的数据）。
- 规划模块目录结构（遵循 `specs/项目结构.md`）。
- 生成 `.ai/config/modules.yaml`，包含模块清单。

## Assert
- 每个功能对应一个模块。
- 模块间依赖关系无循环。

## Artefact
- `.ai/config/modules.yaml`

## Analysis
- 检查是否有模块可以合并（如功能相似）。
- 评估模块独立性，避免过度耦合。

## Adjust
- 如果依赖关系复杂，建议重新组织功能划分。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B7",
    "steps_completed": ["...", "B6"]
  }
  ```
- 提示用户：下一步应执行 B7。

## 干预
无。


## When to Use This Skill
Use this skill when B6_规划模块 is required, such as during phase B tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
