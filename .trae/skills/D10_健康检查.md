# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `D10`。
- `D9` 已完成（SSL证书已配置）。

## 目标
验证应用服务是否已在线上成功运行，通过健康检查接口确认服务可用性。

## Action
- 读取 `.ai/config/deployment.yaml` 中的域名和端口。
- 构造健康检查请求（如 `GET https://your-domain.com/health` 或 `GET http://localhost:8000/health`）。
- 发送 HTTP 请求，设置超时时间（如 30 秒）。
- 检查响应状态码（应为 200）和响应体（如 `{"status":"ok"}`）。
- 若服务有多个实例，逐个检查。
- 记录检查结果和响应时间。

## Assert
- 健康检查请求成功，状态码为 200。
- 响应时间在可接受范围内（如 < 2 秒）。

## Artefact
- 健康检查日志（`.ai/logs/health_check.log`）。

## Analysis
- 如果检查失败，分析原因：服务未启动、端口错误、防火墙阻止、SSL 证书问题。
- 若响应时间过长，提示用户检查服务器负载或数据库性能。

## Adjust
- 若首次失败，等待 10 秒后重试，最多重试 3 次。
- 如果连续失败，回退到 D7 重新启动服务，或提示用户手动检查。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "D11",
    "steps_completed": ["...", "D10"]
  }
  ```
- 提示用户：下一步应执行 D11。

## 干预
如果健康检查反复失败，需用户介入检查服务器状态。


## When to Use This Skill
Use this skill when D10_健康检查 is required, such as during phase D tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
