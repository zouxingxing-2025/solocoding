# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `B9`。
- `B8` 已完成。

## 目标
根据选定的 API 插件和 UI 插件，为每个功能模块生成完整的前后端代码（包括控制器、服务、数据模型、前端组件、路由等）。

## Action
- 读取 `.ai/config/modules.yaml`、`.ai/config/plugins.yaml`、`data-model.json` 以及各模块的 `spec.md`。
- 对于每个模块：
  - 根据 API 插件配置生成后端代码（控制器、服务、数据库访问层、路由）。
  - 根据 UI 插件配置生成前端代码（页面组件、数据绑定、API 调用）。
  - **样式规范**：所有生成的组件必须引用 `docs/design-tokens.css` 中的 CSS 变量，禁止使用硬编码颜色、字体、间距。
  - 如果使用 UI 组件库，必须应用主题定制配置（从 `.ai/config/theme-config.md` 读取），将设计令牌映射到组件库的主题变量。
  - 生成一个全局样式文件（`src/styles/global.css`），添加：`@import url('../../docs/design-tokens.css');` 并覆盖组件库默认样式。
  - 生成对应的单元测试和集成测试（基于功能需求中的验收标准）。
- 生成 API 文档：根据后端代码中的路由和模型，自动生成 OpenAPI/Swagger 文档。
  - 如果使用 FastAPI：框架自动生成 `/docs` 和 `/redoc` 端点，无需额外操作，但需确保依赖已安装。
  - 如果使用 Express.js：集成 `swagger-jsdoc` 和 `swagger-ui-express`，生成 `swagger.json` 并挂载到 `/api-docs`。
  - 如果使用其他框架：生成 `openapi.yaml` 文件，存入 `docs/openapi.yaml`，并提供命令行工具（如 `redoc-cli`）生成 HTML。
- 将生成的代码写入项目目录（遵循 `specs/项目结构.md` 中定义的目录结构）。
- 记录每个模块生成的文件列表。

## Assert
- 每个模块的代码生成完整，无缺失文件。
- 生成的代码符合 `AGENTS.md` 中定义的代码规范（camelCase, 2空格缩进等）。
- 所有 API 接口与前端调用一致。

## Artefact
- 后端代码文件（`backend/src/` 下）
- 前端代码文件（`frontend/pages/` 等）
- 测试文件
- 生成日志 `.ai/logs/codegen.log`
- API 文档（`/docs` 或 `docs/openapi.yaml`）

## Analysis
- 检查是否有代码重复（同一功能在不同模块中重复生成）。
- 验证生成的代码是否遵守胶水编程规则（无重复实现、只做连接与配置）。

## Adjust
- 如果生成失败（如模板缺失），回退到 B7 或 B8 重新选择插件。
- 若检测到代码重复，自动提取公共函数到 `shared/` 目录。
- 如果 Swagger 生成失败且依赖缺失，提示用户安装，但**不中止流程**，记录警告并继续。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "B9.1",
    "steps_completed": ["...", "B9"]
  }
  ```
- 提示用户：下一步应执行 B9.1（文档-代码一致性检查）。

无。
