# 打包部署包

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `F2`。
- `E7` 已完成（集成测试通过）。

## 目标
打包生成最终部署包，包含前端、后端、数据库脚本等所有必要文件，便于部署到生产环境。

## Action
- 读取项目结构，确认所有必要文件已生成。
- 执行打包命令：
  - 前端：`npm run build`
  - 后端：`npm run build` 或 `mvn package`
- 收集打包产物，生成部署包（如 `deploy.zip`）。
- 包含部署说明文档（`deploy/README.md`）。

## Assert
- 部署包生成成功，包含所有必要文件。
- 部署说明文档完整。

## Artefact
- 部署包（`deploy.zip`）。
- 部署说明文档（`deploy/README.md`）。

## Analysis
- 检查打包大小，若过大，考虑优化。

## Adjust
- 若打包失败，修复依赖或配置后重试。

## Advance
- 更新 .ai/进度状态.json：
  ```json
  {
    "current_step": "F3",
    "steps_completed": ["...", "F2"]
  }
  ```
- 提示用户：下一步应执行 F3（上传服务器）。

## 干预
- 无。
