# E9.9_验证修复结果.md

## 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E9.9`。
- `E9.8` 已完成（全量测试已运行）。

## 目标
最终验证缺陷已被完全修复，且修复未引入新的问题。通过人工确认或自动化验证，确保用户可以接受。

## Action
- 读取修复方案和全量测试报告，确认所有测试通过。
- 若修复涉及前端界面或用户交互，生成手动验证步骤清单，指导用户操作。
- 若修复可自动化验证（如 API 响应），则再次执行复现步骤，确认缺陷不再出现。
- 向用户展示验证结果，并提问："请确认缺陷已修复，并且系统功能正常。若满意，请回复'确认'；否则请描述问题。"
- 等待用户回复。

## Assert
- 自动化验证通过（缺陷不再复现）。
- 用户最终确认修复成功。

## Artefact
- 验证报告（`.ai/缺陷报告/当前缺陷/验证报告.md`），包含自动化验证截图或日志、用户确认记录。

## Analysis
- 若用户反馈仍有问题，分析是否根因未完全解决，或修复引入了新的不一致。
- 记录用户确认的时间和环境信息。

## Adjust
- 如果用户确认不通过，将状态回退至 E9.3（定位根因）或 E9.4（生成方案），重新修复。
- 若用户长时间未回复，发送提醒；超时后标记为"待用户确认"。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E9.10",
    "steps_completed": ["...", "E9.9"]
  }
  ```
- 提示用户：下一步应执行 E9.10（生成修复报告并更新文档）。

## 干预
用户需确认修复结果。
