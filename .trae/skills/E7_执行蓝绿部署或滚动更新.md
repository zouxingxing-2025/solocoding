# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E7`。
- `E6` 已完成（受影响模块已重新生成）。

## 目标
将更新后的模块部署到生产环境，采用蓝绿部署或滚动更新策略，确保服务不中断。

## Action
- 读取 `.ai/config/deployment.yaml` 中的部署策略（蓝绿部署或滚动更新）。
- 若使用蓝绿部署：
  - 将新版本部署到备用环境（绿环境）。
  - 在绿环境中运行冒烟测试（健康检查、核心API测试）。
  - 切换负载均衡器流量从蓝环境到绿环境。
  - 保留蓝环境作为回滚备份。
- 若使用滚动更新（如 Kubernetes、Docker Swarm）：
  - 逐步替换旧实例，每次更新一个或一定比例的实例。
  - 监控更新过程中的错误率和响应时间。
  - 若出现问题，自动停止更新并回滚。
- 记录部署操作日志（`.ai/logs/deploy_update.log`）。
- 更新 `.ai/进度状态.json` 中的部署版本号和时间戳。

## Assert
- 新版本已成功部署且服务正常。
- 旧版本可随时回滚（蓝绿部署）或已安全替换（滚动更新）。

## Artefact
- 部署日志。
- 更新后的进度状态。

## Analysis
- 检查部署过程中是否有错误（如容器启动失败、健康检查超时）。
- 若有异常，自动触发回滚。

## Adjust
- 如果蓝绿部署切换失败，自动回切到旧环境。
- 若滚动更新失败，暂停更新并回滚已更新的实例。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E8",
    "steps_completed": ["...", "E7"]
  }
  ```
- 提示用户：下一步应执行 E8（更新核心契约）。

## 干预
若部署过程中出现严重问题且无法自动回滚，需用户手动介入。


## When to Use This Skill
Use this skill when E7_执行蓝绿部署或滚动更新 is required, such as during phase E tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
