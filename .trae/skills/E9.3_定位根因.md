# 前置条件
- `.ai/进度状态.json` 中的 `current_step` 必须为 `E9.3`。
- `E9.2` 已完成（缺陷已成功复现）。

## 目标
分析复现日志和相关代码，定位缺陷的根本原因（如逻辑错误、边界条件、数据竞争、配置问题等）。

## Action
- 读取复现日志（`.ai/缺陷报告/当前缺陷/复现日志.log`）和相关模块的代码。
- 根据复现步骤，追踪执行路径：
  - 后端：查看请求日志、数据库查询、异常堆栈。
  - 前端：检查浏览器控制台错误、网络请求、状态变化。
- 使用静态分析工具或手动排查，找出导致异常的具体代码行或配置项。
- 输出根因分析报告（`.ai/缺陷报告/当前缺陷/根因分析.md`），包含：
  - 缺陷触发条件。
  - 涉及的文件和函数。
  - 根本原因描述（如"空指针异常"、"SQL 注入风险"、"缓存未失效"）。
- 如果匹配到全局缺陷模式库中的已知模式，引用该模式。

## Assert
- 根因分析报告清晰指出问题所在。
- 若无法定位，标记为"需人工分析"。

## Artefact
- 根因分析报告（`.ai/缺陷报告/当前缺陷/根因分析.md`）。

## Analysis
- 检查缺陷是否由多个因素共同导致（如并发、环境依赖）。
- 评估影响范围（是否影响其他模块或数据）。

## Adjust
- 如果根因不明确，建议添加更详细的日志或使用调试工具，返回 E9.2 再次复现。
- 若涉及第三方库，检查是否有已知 bug 或版本兼容性问题。

## Advance
- 更新 `.ai/进度状态.json`：
  ```json
  {
    "current_step": "E9.4",
    "steps_completed": ["...", "E9.3"]
  }
  ```
- 提示用户：下一步应执行 E9.4（生成修复方案）。

## 干预
若无法自动定位根因，需用户提供更多信息或手动分析。


## When to Use This Skill
Use this skill when E9.3_定位根因 is required, such as during phase E tasks.

## Not For
- This skill does not handle other phases or unrelated tasks.
